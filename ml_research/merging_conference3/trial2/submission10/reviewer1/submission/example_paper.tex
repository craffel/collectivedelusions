\section{Experimental and Hyperparameter Details}
\label{app:hyperparameters}

In this appendix, we provide additional details regarding our training and optimization pipeline, model architectures, and cluster resources to ensure absolute reproducibility.

\subsection{Task Expert Architectures and Pre-training}
As described in Section~\ref{sub:setup}, we use the pre-trained CLIP ViT-B/32 visual encoder \cite{clip} as our shared base model $\theta_0$. The visual encoder consists of a 12-layer Vision Transformer (ViT) with patch size $32 \times 32$. The model processes inputs of shape $224 \times 224 \times 3$ pixels.

For each task (MNIST, FashionMNIST, CIFAR-10, SVHN), we initialize a linear classification head whose input dimension is $512$ (the hidden dimension of the CLIP visual encoder) and whose output dimension matches the number of task classes (10 classes for all four tasks).

\subsection{Expert Head Fine-Tuning Hyperparameters}
During the head fine-tuning phase, the parameters of the CLIP visual encoder are frozen, and only the classification heads are trained. The hyperparameters are configured as follows:
\begin{itemize}
    \item \textbf{Optimizer}: AdamW \cite{adamw}
    \item \textbf{Learning Rate}: $10^{-3}$
    \item \textbf{Weight Decay}: $10^{-4}$
    \item \textbf{Batch Size}: $32$
    \item \textbf{Epochs}: 5
    \item \textbf{Loss Function}: Cross-Entropy Loss
\end{itemize}

\subsection{Test-Time Entropy Minimization Details}
For both AdaMerging and Flat-AdaMerging, test-time adaptation is performed on a combined calibration batch $X$ of size 256 (64 unlabeled images sampled from each of the 4 task splits). The prediction entropy is minimized using two independent optimizers:
\begin{itemize}
    \item \textbf{1+1 Evolution Strategy (1+1 ES)}:
    \begin{itemize}
        \item Total Steps: 500
        \item Initial Scaling Coefficients: $0.3$ (layer-wise or task-wise)
        \item Mutation Noise Scale ($\sigma$): $0.01$
        \item Selection Rule: Success-based mutation adjustment (standard 1+1 ES update rule)
    \end{itemize}
    \item \textbf{Adam Gradient Descent (Adam GD)}:
    \begin{itemize}
        \item Total Steps: 200
        \item Initial Scaling Coefficients: $0.3$ (initialized as leaf tensors to prevent non-leaf computational graph warnings)
        \item Learning Rate ($\eta$): $10^{-2}$
        \item Optimizer Parameters: $\beta_1 = 0.9, \beta_2 = 0.999, \epsilon = 10^{-8}$
    \end{itemize}
\end{itemize}

\subsection{Computing Infrastructure and Run-Times}
All experiments were conducted on a high-performance computing cluster. Independent expert fine-tuning and weight merging were executed using Python 3.12 and PyTorch 2.12.0+cu130. 

By profiling and optimizing the model merging pipeline to run entirely on the GPU (keeping task vectors and base model parameters in CUDA memory), we reduced the total execution time of the entire pipeline---including expert fine-tuning, coefficient optimization across 3 seeds, noise sweeps, and Linear CKA representation measurements---from over 35 minutes down to approximately \textbf{2.5 minutes per seed}.
