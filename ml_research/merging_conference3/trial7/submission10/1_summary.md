# Summary of the Submission

## 1. Context and Problem Statement
Deploying multiple specialized parameter-efficient task adapters (like Low-Rank Adaptation (LoRA) modules) concurrently on resource-constrained edge devices (e.g., mobile phones, IoT devices, embedded CPUs, and microcontrollers) is highly challenging.
* **Static model-merging** methods (e.g., Task Arithmetic, TIES-Merging, DARE) statically average the expert parameters before deployment. While this adds zero parameter or activation memory overhead, it is highly prone to **heterogeneity collapse** when processing a mixed-task batch/stream, as a single, static set of weights cannot accommodate conflicting activation patterns.
* **Dynamic model-merging** methods (e.g., Parameter-Free Subspace Routing (PFSR), Micro-Batch Homogenization (MBH)) dynamically switch or route activations. However, state-of-the-art MBH partitions heterogeneous batches on-the-fly and processes them separately, requiring $G \le K$ sequential forward passes of the massive pre-trained backbone. This introduces a linear latency penalty ($O(K)$ scaling) that causes significant latency spikes, which is unacceptable for interactive edge deployment.
* **Temporal Routing Paradox:** Traditional dynamic routers require late-stage (penultimate) representation features to compute routing coefficients. Doing so forces the system to run the entire backbone once just to get routing coefficients, and then a second time to execute task-specific adapters, doubling the execution latency.

## 2. Proposed Method (SPS-ZCA)
To resolve these bottlenecks, the paper proposes **SPS-ZCA** (Single-Pass Sample-Wise Routing with Zero-Shot Centroid Alignment), a training-free, extremely fast, and robust dynamic model-merging framework:
1. **Single-Pass Activation-Space Dynamic Blending (SPS):** Instead of splitting heterogeneous batches and executing multiple sequential backbone passes, SPS computes the frozen base representation once for the entire batch and blends the activations of the parallel, task-specific expert adapters on-the-fly using sample-wise, un-averaged routing coefficients. This achieves $O(1)$ constant-time latency.
2. **Zero-Shot Centroid Alignment (ZCA):** To resolve the temporal routing paradox, ZCA pre-computes robust, head-independent task centroids in the pre-trained backbone's early representation space (Layer 3) during a one-time offline calibration phase (using only 64 samples per task). During inference, inputs are routed in this shared, adapter-free early space, which is highly task-separable ($\text{FSC} = 47.50$).
3. **Resolving the Early-Layer Paradox:** The authors keep Layers 1--3 of the Vision Transformer backbone shared and frozen (task-agnostic with no adapters), and insert LoRA adapters strictly in mid-to-late layers (Layers 4--$L$). This guarantees 100% mathematical consistency with zero train-inference mismatch, as no early-stage adapters need to be executed before routing.
4. **Calibration and Robustness Upgrades:**
    * **Unit-Norm Calibration (UNC):** Forces representations and centroids onto a unit hypersphere (equivalent to cosine similarity), neutralizing cross-expert scale imbalances.
    * **Intra-Task Dispersion Calibration (IDC):** Normalizes similarity coordinates by their expected in-distribution dispersion scale, preventing compact task manifolds from statistically dominating over dispersed domains (e.g., SVHN).
    * **OOD GMM Coordinate Shield:** Fits a diagonal Gaussian Mixture Model (GMM) with $M=2$ components over the calibration coordinates to filter and reject out-of-distribution (OOD) queries, triggering a clean, domain-appropriate fallback (e.g., falling back to the frozen base model for text or raising an OOD flag for vision) rather than misrouting.
    * **Adaptive Entropy-Dependent Temperature Scaling:** Dynamically scales Softmax routing temperature as a function of coordinate Shannon entropy to enforce sharp routing for clear inputs while gracefully blending activations for ambiguous, borderline states.

## 3. Core Contributions and Results
* **Perfect Expert Ceiling Recovery:** On mixed-task streaming benchmarks, SPS-ZCA achieves a Joint Mean accuracy of **79.80%** (vision sandbox) and **76.14%** (physical PyTorch model on real images), recovering **100.0% of the Expert Ceiling** and outperforming PFSR+MBH SOTA by **+3.66%** absolute accuracy.
* **Systems Speedups:** 
    * **Verified Physical Speedup:** Under small batch scales ($B=16$), where sequential launch overheads dominate, uncompiled PyTorch using Vectorized Scatter-Gather (`SPS-VSG`) achieves a verified physical **1.17$\times$ wall-clock speedup** out of the box (16.63 ms vs. MBH's 19.42 ms).
    * **Hardware-Aware Cost Model Projection:** Projects a **3.90$\times$ speedup** (slashing cost from 776.4 ms to 199.0 ms) under a compiled layout that avoids dynamic framework memory indexing overheads.
    * **ONNX Runtime CustomOp Raspberry Pi Benchmark:** Benchmarking the custom C++ operator natively on a physical Raspberry Pi 4 CPU achieves a physical **3.91$\times$ wall-clock speedup** at $B=1$ (22.6 ms vs. MBH's 88.4 ms) and **3.61$\times$ speedup** at $B=256$ (215.1 ms vs. MBH's 776.4 ms), completely closing the "serving gap".
* **Text Modality Generalizability:** Validates text sequences on GPT-2 adapters across Legal, Medical, and Coding domains, achieving 98.50% routing accuracy, 91.83% joint classification accuracy, and near-perfect perplexity preservation (12.18 Joint Mean vs. 12.15 ceiling, compared to Uniform weight merging's catastrophic 84.50 perplexity).
* **High-Density Scaling sweeps:** Validates robustness up to $K=128$ task experts, indicating ZCA routing accuracy remains perfectly stable at 100% for $K \le 16$, and degrades gracefully to 96.80% at $K=64$ and 88.50% at $K=128$.
