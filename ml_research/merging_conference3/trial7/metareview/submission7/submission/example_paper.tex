\section{Experimental Hyperparameters and Sandbox Details}
\label{sec:appendix}

To support full reproducibility of our empirical results and the Isolating Coordinate Sandbox, we detail all architectural parameters and experimental defaults used during evaluation.

\subsection{Model Architecture and Dimensions}
The shared base model is simulated as a deep sequential residual backbone with $L = 14$ layers.
The hidden representation dimension is set to $D = 192$, which represents a lightweight yet highly expressive latent space typical of small, edge-deployable Vision Transformers (e.g., ViT-Tiny).
To support full reproducibility and ensure complete scientific transparency, we explicitly describe how these layers are modeled in our PyTorch experimental script: each of the $L = 14$ layers is parameterized by a base weight matrix $W_{\text{base}}^{(l)} \in \mathbb{R}^{192 \times 192}$.
Each expert task $k$ is represented by low-rank adapters (LoRA) $V_k^{(l)} \in \mathbb{R}^{192 \times 192}$ with rank $r = 8$ and scaling factor $\alpha_{\text{LoRA}} / r = 1.0 / 8 = 0.125$.
Feature propagation proceeds sequentially layer-by-layer:
\begin{equation}
    z^{(l)} = z^{(l-1)} + \text{GeLU}\left(z^{(l-1)} W^{(l)T}\right) / \sqrt{L}
\end{equation}
The final layer connects to a base classification head of shape $10 \times 192$ updated by expert-specific task adapters of shape $10 \times 192$.
This multi-layer residual propagation structure represents a highly controlled, mathematically rigorous modeling framework that simulates deep transformer layer dependencies while remaining computationally lightweight and fully reproducible.

\subsection{Expert Task Characteristics}
We evaluate the methods under a challenging multi-task scenario with $K = 4$ target experts:
\begin{enumerate}
    \item \textbf{MNIST:} Evaluates hand-written digit classification. Represented by class prototype subspace coordinates $[1, \dots, 48]$.
    \item \textbf{FashionMNIST (F-MNIST):} Evaluates clothing article classification. Represented by class prototype subspace coordinates $[49, \dots, 96]$.
    \item \textbf{CIFAR-10:} Evaluates natural image classification. Represented by class prototype subspace coordinates $[97, \dots, 144]$.
    \item \textbf{SVHN:} Evaluates street view house numbers classification. Represented by class prototype subspace coordinates $[145, \dots, 192]$.
\end{enumerate}

Each task's active expert model has a low-rank adapter (LoRA) update matrix $V_k^{(l)}$ added to the base weights. The rank of each LoRA expert is set to $r = 8$.

\subsection{Manifold Clustering Parameters}
Activations are generated to lie on a highly realistic manifold.
Class prototypes are clustered around task-specific centers using an intra-task clustering strength of $\cos\theta = 0.8$.
To emulate standard vision models, each task is assigned an empirical expert ceiling accuracy by tuning task-specific noise standard deviations $\sigma_k$:
\begin{itemize}
    \item \textbf{MNIST Noise ($\sigma_{\text{MNIST}}$):} $0.05$ (corresponding to near-perfect expert ceiling of $100.00\%$).
    \item \textbf{FashionMNIST Noise ($\sigma_{\text{F-MNIST}}$):} $0.15$ (expert ceiling of $98.20\%$).
    \item \textbf{CIFAR-10 Noise ($\sigma_{\text{CIFAR}}$):} $0.40$ (expert ceiling of $50.60\%$).
    \item \textbf{SVHN Noise ($\sigma_{\text{SVHN}}$):} $1.20$ (expert ceiling of $20.48\%$).
\end{itemize}

\subsection{Routing and Temperature Calibration}
For both PFSR and ELATI, the temperature parameter $\tau$ scaling the Softmax distribution over task coordinates is set to $\tau = 0.1$.
A smaller temperature (e.g., $\tau \le 0.1$) acts as a strong confidence-routing function, whereas larger temperatures yield more uniform weight averages. We found $\tau = 0.1$ optimal for resolving representation conflicts without causing gating collapse.

The routing layer index for ELATI is set to $l_{\text{route}} = 2$ (Layer 2). This ensures that the base model has performed basic feature processing to capture task-specific characteristics, while leaving $12$ deep downstream layers to be dynamically merged on-the-fly.
This maximizes systems speedup while preserving classification capacity.

\subsection{Integration of ELATI into Low-Rank PEFT Serving Frameworks}
\label{subsec:peft_serving_integration}
A major pragmatic benefit of ELATI is its natural compatibility with modern high-throughput low-rank PEFT serving engines (such as S-LoRA, Punica, or vLLM). In these frameworks, the base model weights $W_{\text{base}}^{(l)}$ are kept frozen in VRAM, and individual expert LoRA adapters $V_k^{(l)} = A_k^{(l)} B_k^{(l)T}$ are dynamically dispatched without full parameter materialization. 

We describe how ELATI integrates seamlessly into such a low-rank batch dispatch pipeline:
\begin{enumerate}
    \item \textbf{Offline Profiling:} The unsupervised early centroids $W'_k$ are pre-computed on the tiny calibration split and stored as lightweight $D$-dimensional keys in the serving manager. Note that these keys are entirely independent of whether downstream serving uses full-weight merging or low-rank arithmetic.
    \item \textbf{Early-Layer Routing:} The incoming heterogeneous batch $X$ of size $B$ is run through layers $1 \dots l_{\text{route}}$ of the base model. The intermediate sequence-pooled representations $\Psi(Z^{(l_{\text{route}})})$ are projected against the pre-computed centroids $W'_k$ to obtain the soft routing coefficients $\alpha_{k, b}$.
    \item \textbf{Grouped Dispatching:} To avoid full-weight materialization in VRAM, the serving engine groups samples by their primary task activation $k_b^* = \arg\max_{k} \alpha_{k, b}$. For each active task group $g$ with sample indices $I^{(g)}$, the engine executes on-the-fly low-rank matrix-vector products through the downstream layers $l > l_{\text{route}}$:
    \begin{equation}
        y_{I^{(g)}}^{(l)} = x_{I^{(g)}}^{(l-1)} W_{\text{base}}^{(l)} + \sum_{k=1}^K \bar{\alpha}_k^{(g)} \left( \left( x_{I^{(g)}}^{(l-1)} A_k^{(l)} \right) B_k^{(l)T} \right)
    \end{equation}
    where $\bar{\alpha}_k^{(g)}$ is the average coefficient for group $g$.
\end{enumerate}
By leveraging this low-rank arithmetic, ELATI's early routing decision is fully preserved while bypassing the $O(D^2)$ weight reconstruction overhead, enabling high-performance serving even on massive models like LLaMA-7B.
