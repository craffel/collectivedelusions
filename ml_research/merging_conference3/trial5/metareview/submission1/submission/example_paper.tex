\section{Extended Mathematical Formulations}

\subsection{Fisher Information Matrix (FIM) Trace Derivation}
In this section, we derive the FIM trace calculation of the pre-trained base model $\theta_0$ as a spatial curvature metric. Let $p_{\theta_0}(y|x)$ represent the conditional probability distribution of the output $y$ given the input $x$ parameterized by weights $\theta_0$. For a specific layer $l \in \{1, \dots, L\}$ with $d_l$ parameters, the FIM block $F^{(l)} \in \mathbb{R}^{d_l \times d_l}$ is defined as:
\begin{equation}
F^{(l)} = \mathbb{E}_{x \sim \mathcal{D}, y \sim p_{\theta_0}(y|x)} \left[ \nabla_{\theta_0^{(l)}} \log p_{\theta_0}(y|x) \nabla_{\theta_0^{(l)}} \log p_{\theta_0}(y|x)^\top \right]
\end{equation}
Taking the trace of this matrix block, we obtain:
\begin{align}
\text{Tr}(F^{(l)}) &= \text{Tr}\left( \mathbb{E}_{x \sim \mathcal{D}, y \sim p_{\theta_0}(y|x)} \left[ \nabla_{\theta_0^{(l)}} \log p_{\theta_0}(y|x) \nabla_{\theta_0^{(l)}} \log p_{\theta_0}(y|x)^\top \right] \right) \nonumber \\
&= \mathbb{E}_{x \sim \mathcal{D}, y \sim p_{\theta_0}(y|x)} \left[ \text{Tr}\left( \nabla_{\theta_0^{(l)}} \log p_{\theta_0}(y|x) \nabla_{\theta_0^{(l)}} \log p_{\theta_0}(y|x)^\top \right) \right] \nonumber \\
&= \mathbb{E}_{x \sim \mathcal{D}, y \sim p_{\theta_0}(y|x)} \left[ \|\nabla_{\theta_0^{(l)}} \log p_{\theta_0}(y|x)\|_2^2 \right]
\end{align}
Normalizing this trace by the layer parameter dimension $d_l$ to ensure comparability across layers of varying shapes, we arrive at the empirical base curvature metric $c_l$:
\begin{equation}
c_l = \frac{1}{|D_{\text{cal}}|} \sum_{x \in D_{\text{cal}}} \mathbb{E}_{y \sim p_{\theta_0}(y|x)} \left[ \frac{1}{d_l} \|\nabla_{\theta_0^{(l)}} \log p_{\theta_0}(y|x)\|_2^2 \right]
\end{equation}
This formulation demonstrates that $c_l$ is exactly proportional to the average squared norm of the gradients of the model log-likelihood with respect to layer $l$\rq s parameters. This metric is a direct second-order measure of the sensitivity of layer $l$\rq s functional representations under parameter perturbations.

\subsection{Coupled Model II Landscape Equations}
We detail the exact mathematical formulation of the Coupled Model II Landscape. For each task $k \in \{1, \dots, K\}$ and layer $l \in \{1, \dots, L\}$, let $A_k^{(l)}$ denote the ground-truth quadratic sensitivity of layer $l$ under task $k$.
The multi-task accuracy on task $k$ under merging coefficients $\boldsymbol{\lambda}$ is simulated as:
\begin{equation}
\text{Acc}_k(\boldsymbol{\lambda}) = \text{Acc}_k^{\text{max}} - (\boldsymbol{\lambda}_k - \boldsymbol{\lambda}_k^*)^{\top} \boldsymbol{\Sigma}_k^{-1} (\boldsymbol{\lambda}_k - \boldsymbol{\lambda}_k^*)
\end{equation}
where $\boldsymbol{\lambda}_k \in \mathbb{R}^L$ is the vector of coefficients for task $k$ across depth, and $\boldsymbol{\lambda}_k^* \in \mathbb{R}^L$ is the ground-truth optimal coefficient trajectory for task $k$, which is a smooth function representing depth-dependent scaling.
The inverse spatial covariance matrix $\boldsymbol{\Sigma}_k^{-1}$ couples adjacent layers. The elements of $\boldsymbol{\Sigma}_k$ are modeled as:
\begin{equation}
(\Sigma_k)_{i, j} = \sqrt{\frac{1}{A_k^{(i)} A_k^{(j)}}} \cdot \rho^{|i-j|}
\end{equation}
where $\rho \in (0, 1)$ is the spatial coupling coefficient (set to $0.5$ in our experiments). 
Taking the inverse $\boldsymbol{\Sigma}_k^{-1}$ introduces a tridiagonal-like structure that heavily penalizes large differences between adjacent elements:
\begin{equation}
(\boldsymbol{\lambda}_k - \boldsymbol{\lambda}_k^*)^{\top} \boldsymbol{\Sigma}_k^{-1} (\boldsymbol{\lambda}_k - \boldsymbol{\lambda}_k^*) \propto \sum_{l=1}^L A_k^{(l)} (\lambda_{k, l} - \lambda_{k, l}^*)^2 + \sum_{l=2}^L \sqrt{A_k^{(l)} A_k^{(l-1)}} \omega (\lambda_{k, l} - \lambda_{k, l-1})^2
\end{equation}
where $\omega$ is a coupling weight. This equation shows that any high-frequency, uncoordinated oscillations in $\boldsymbol{\lambda}$ will severely inflate the quadratic error and collapse the simulated accuracy. This formulation provides a mathematically rigorous analog to the physical laws of representation routing in deep neural networks.

\subsection{Practical PyTorch Recipe for Real-World Transformer Deployment}
To demonstrate the physical viability and ease of scaling RCR-Merge to actual deep networks, we provide a complete, standalone PyTorch recipe. This turnkey implementation shows: (1) how to estimate the diagonal FIM trace offline for a ViT model, (2) how to dynamically compute GNB at step 0, and (3) how to perform joint dual-regularized test-time adaptation.

import torch
import torch.nn as nn
from transformers import ViTForImageClassification

# 1. Base Curvature Estimation (FIM Diagonal Trace)
def estimate_base_curvature(model, calibration_loader, device="cuda"):
    model.eval()
    model.to(device)
    # Track gradient squared trace per block/layer
    grad_traces = {l: 0.0 for l in range(12)}  # ViT-B has 12 blocks
    param_counts = {l: 0 for l in range(12)}
    
    for x, _ in calibration_loader:
        x = x.to(device)
        # Compute forward pass and log-likelihood
        outputs = model(x)
        log_probs = torch.log_softmax(outputs.logits, dim=-1)
        
        # Sample fake target labels y ~ p_theta(y|x) for Fisher definition
        probs = torch.exp(log_probs)
        y_sampled = torch.multinomial(probs.view(-1, probs.size(-1)), 1).squeeze()
        
        loss = nn.CrossEntropyLoss()(outputs.logits, y_sampled)
        model.zero_grad()
        loss.backward()
        
        # Aggregate squared gradient norms per transformer block
        with torch.no_grad():
            for name, param in model.named_parameters():
                if "encoder.layer" in name and param.grad is not None:
                    # Extract block index
                    l = int(name.split("encoder.layer.")[1].split(".")[0])
                    grad_traces[l] += torch.sum(param.grad ** 2).item()
                    param_counts[l] += param.numel()
                    
    # Normalize by dataset size and layer dimension
    c = torch.tensor([grad_traces[l] / (len(calibration_loader.dataset) * param_counts[l])
                      for l in range(12)], dtype=torch.float32)
    return c / torch.mean(c)  # Normalize metric tensor trace

# 2. RCR-Merge Joint Test-Time Adaptation with GNB
class RCRMergeTTA:
    def __init__(self, experts, base_model, c_l, alpha_gnb=0.7, gamma_anchor=0.1):
        self.experts = experts  # List of K expert state_dicts
        self.base_model = base_model  # Shared backbone model
        self.c_l = c_l  # Base curvature vector (12,)
        self.alpha_gnb = alpha_gnb
        self.gamma_anchor = gamma_anchor
        self.device = next(base_model.parameters()).device
        
        # Initialize learnable merging coefficients (K task experts, L layers)
        self.K = len(experts)
        self.L = 12
        self.coeffs = nn.Parameter(torch.ones((self.K, self.L), device=self.device) * 0.3)
        self.optimizer = torch.optim.Adam([self.coeffs], lr=0.01)
        
        # Pre-calculate RCR-TV metric tensor layer coupling: sqrt(c_l * c_{l-1})
        self.weight_l = torch.sqrt(self.c_l[1:] * self.c_l[:-1]).to(self.device)
        
    def _apply_merged_weights(self):
        # Build task vectors and interpolate
        base_sd = self.base_model.state_dict()
        merged_sd = {}
        for name in base_sd.keys():
            if "encoder.layer" in name:
                l = int(name.split("encoder.layer.")[1].split(".")[0])
                # Merge task vectors linearly with layer-wise coefficients
                merged_param = base_sd[name].clone().float()
                for k in range(self.K):
                    v_k = self.experts[k][name].float() - base_sd[name].float()
                    merged_param += self.coeffs[k, l] * v_k
                merged_sd[name] = merged_param
            else:
                merged_sd[name] = base_sd[name]
        self.base_model.load_state_dict(merged_sd)

    def compute_losses(self, x):
        # 1. Entropy TTA Loss
        self._apply_merged_weights()
        outputs = self.base_model(x)
        probs = torch.softmax(outputs.logits, dim=-1)
        tta_loss = -torch.mean(torch.sum(probs * torch.log(probs + 1e-8), dim=-1))
        
        # 2. Curvature-Weighted Spatial TV Penalty
        diff_sq = (self.coeffs[:, 1:] - self.coeffs[:, :-1]) ** 2
        rcr_penalty = torch.sum(self.weight_l * diff_sq)
        
        # 3. Absolute Coordinate Anchor Penalty
        anchor_penalty = torch.sum((self.coeffs - 0.3) ** 2)
        
        return tta_loss, rcr_penalty, anchor_penalty

    def run_gnb_and_initialize(self, x_init):
        # Calculate TTA loss gradient norm at lambda_0 (0.3)
        tta_loss, _, _ = self.compute_losses(x_init)
        self.optimizer.zero_grad()
        tta_loss.backward()
        norm_grad_tta = torch.norm(self.coeffs.grad).item()
        
        # Calculate regularizer gradient norm at pert_state (alternating pattern)
        delta_gnb = 0.05
        pert_pattern = torch.tensor([(-1.0)**l for l in range(self.L)], device=self.device)
        with torch.no_grad():
            self.coeffs.copy_(0.3 + delta_gnb * pert_pattern.unsqueeze(0).repeat(self.K, 1))
            
        _, rcr_penalty, _ = self.compute_losses(x_init)
        self.optimizer.zero_grad()
        rcr_penalty.backward()
        norm_grad_rcr = torch.norm(self.coeffs.grad).item()
        
        # Compute dynamic unsupervised beta and reset coefficients to 0.3
        self.beta = self.alpha_gnb * norm_grad_tta / norm_grad_rcr
        with torch.no_grad():
            self.coeffs.copy_(torch.ones((self.K, self.L), device=self.device) * 0.3)
        print(f"GNB Initialized: Beta = {self.beta:.4f}")

    def adapt_step(self, x_batch):
        tta_loss, rcr_penalty, anchor_penalty = self.compute_losses(x_batch)
        # Joint optimization objective: TTA + dynamic_beta * TV + anchor
        total_loss = tta_loss + self.beta * rcr_penalty + self.gamma_anchor * anchor_penalty
        
        self.optimizer.zero_grad()
        total_loss.backward()
        self.optimizer.step()
        with torch.no_grad():
            self.coeffs.clamp_(0.0, 1.0)  # Maintain coefficient feasibility
