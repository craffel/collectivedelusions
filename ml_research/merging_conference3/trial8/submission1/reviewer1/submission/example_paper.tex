\section{Proofs and Mathematical Derivations}

\subsection{Poincaré Ball Primitives and Curvature Properties}
In this appendix, we outline the mathematical properties of the Poincaré Ball model $\mathbb{D}_c^D$ with curvature $c > 0$. The Riemannian metric is given by $g_{\mathbf{x}} = \lambda_{\mathbf{x}}^2 g_{\text{Eucl}}$, where $\lambda_{\mathbf{x}} = \frac{2}{1 - c\|\mathbf{x}\|_2^2}$ is the conformal factor. For any $\mathbf{x}, \mathbf{y} \in \mathbb{D}_c^D$, the geodesic distance is:
\begin{equation}
d_{\mathbb{D}}^c(\mathbf{x}, \mathbf{y}) = \frac{2}{\sqrt{c}} \text{artanh}\left( \sqrt{c} \|\!-\mathbf{x} \oplus_c \mathbf{y}\|_2 \right).
\end{equation}
This distance expands exponentially as points approach the boundary of the ball $\|\mathbf{x}\|_2 \to 1/\sqrt{c}$, which mathematically scales the available representation area, naturally resolving representation crowding.

\subsection{Möbius Addition Properties}
Möbius addition is non-commutative and non-associative, but it satisfies the following properties:
\begin{align}
\mathbf{x} \oplus_c \mathbf{0} &= \mathbf{0} \oplus_c \mathbf{x} = \mathbf{x} \\
\mathbf{x} \oplus_c (-\mathbf{x}) &= \mathbf{0} \\
-\mathbf{x} \oplus_c (\mathbf{x} \oplus_c \mathbf{y}) &= \mathbf{y}.
\end{align}
These properties are critical for ensuring that projecting expert updates into and out of the Poincaré Ball preserves directional alignment while allowing non-linear activation scaling and ensembling.

\subsection{Einstein Midpoint in Klein Space}
The hyperbolic centroid (or Fréchet mean) of a set of points in the Poincaré Ball does not have a closed-form solution in the Poincaré metric itself. However, by mapping to the Klein model $\mathbb{K}_c^D$, we can exploit the projective flat properties of Klein space. The mapping from the Poincaré Ball $\mathbb{D}_c^D$ to the Klein model $\mathbb{K}_c^D$ is given by:
\begin{equation}
\mathbf{z}^{Klein} = \frac{2 \mathbf{z}}{1 + c \|\mathbf{z}\|_2^2}.
\end{equation}
Once in Klein coordinates, the Fréchet mean corresponds exactly to the Einstein midpoint, which is a weighted average of the coordinates weighted by their Lorentz factors $\gamma$:
\begin{equation}
\boldsymbol{\mu}^{Klein} = \frac{\sum_{s} \gamma_s \mathbf{z}_s^{Klein}}{\sum_{s} \gamma_s}, \quad \text{where } \gamma_s = \frac{1}{\sqrt{1 - c \|\mathbf{z}_s^{Klein}\|_2^2}}.
\end{equation}
We then project this Einstein midpoint back to the Poincaré Ball coordinates to obtain the mathematically optimal hyperbolic centroid reference $\boldsymbol{\mu}$:
\begin{equation}
\boldsymbol{\mu} = \frac{\boldsymbol{\mu}^{Klein}}{1 + \sqrt{1 - c \|\boldsymbol{\mu}^{Klein}\|_2^2}}.
\end{equation}
This formulation ensures that the centroid is robust to outliers and represents the mathematically true center of gravity in a negatively curved space.

\section{Actionable Real-World Evaluation Blueprint on Foundation Models}
\label{app:realworld}

While the Analytical Coordinate Sandbox serves as an excellent theoretical proof of concept for dynamic model ensembling, we outline a complete, actionable, and ready-to-use experimental blueprint to validate \textbf{HyperMerge} on physical pre-trained foundation models (e.g., ViT-B/16, CLIP, or RoBERTa-base) on real multi-task datasets.

\subsection{Experimental Protocol for Vision-Language Models (CLIP)}
To evaluate HyperMerge on a real-world multi-task computer vision serving pipeline, we propose adaptation of a pre-trained \textbf{CLIP ViT-B/32} backbone across four diverse tasks from the Visual Decathlon benchmark:
\begin{enumerate}
    \item \textbf{CIFAR-100} (general image classification)
    \item \textbf{SVHN} (digit recognition)
    \item \textbf{Oxford Flowers} (fine-grained natural images)
    \item \textbf{Resisc45} (remote sensing satellite imagery)
\end{enumerate}

For each task, we fine-tune a dedicated LoRA adapter ($r=8$) targeting the query, key, and value projection matrices in all 12 self-attention layers of the CLIP image encoder. The backbone weights are frozen. After fine-tuning, task centroids in hyperbolic space are computed using HCA from 32 support images per task, projecting Layer 0 patch embeddings to the Poincaré Ball.

During serving, a heterogeneous stream containing mixed images from all four tasks is fed to the model. At Layer 0, the patch embedding is mapped to the Poincaré Ball to compute dynamic routing weights using hyperbolic geodesic distance to the pre-computed HCA centroids. At attention layers 10 and 11, the individual experts are evaluated in parallel, and their activation-space updates are mapped to the Poincaré Ball, blended symmetrically using BKSB (Einstein midpoint in Klein coordinates), mapped back to Euclidean space, and added residually to the base transformer activations.

\subsection{Production-Ready PyTorch Implementation}
Below, we provide a complete, production-grade PyTorch implementation of the HyperMerge ensembling module, which handles both sequence-level token representations and batched inputs. It can be easily integrated into any standard transformer backbone from the Hugging Face \texttt{transformers} library or parameter-efficient fine-tuning via \texttt{peft}.

\begin{verbatim}
import torch
import torch.nn as nn

class HyperMergeModule(nn.Module):
    def __init__(self, dim, num_tasks, c=0.1, tau=0.05):
        super().__init__()
        self.dim = dim
        self.num_tasks = num_tasks
        self.c = c
        self.tau = tau
        # Learnable/calibrated task centroids in Hyperbolic space
        self.centroids = nn.Parameter(torch.randn(num_tasks, dim) * 0.01)
        
    def exp_map(self, h):
        norm = torch.norm(h, dim=-1, keepdim=True) + 1e-8
        sqrt_c = self.c ** 0.5
        return torch.tanh(sqrt_c * norm) / (sqrt_c * norm) * h

    def log_map(self, z):
        norm = torch.norm(z, dim=-1, keepdim=True) + 1e-8
        sqrt_c = self.c ** 0.5
        val = torch.clamp(sqrt_c * norm, 0.0, 1.0 - 1e-7)
        artanh_val = 0.5 * torch.log((1.0 + val) / (1.0 - val))
        return artanh_val / val * z
        
    def forward(self, h_base, expert_updates, routing_features):
        # h_base: [B, seq_len, dim] - Euclidean base model output
        # expert_updates: list of K tensors of shape [B, seq_len, dim]
        # routing_features: [B, dim] - Layer 0 query embeddings
        B = h_base.shape[0]
        
        # 1. Project routing features & compute routing weights
        z_route = self.exp_map(routing_features) # [B, dim]
        u = torch.zeros(B, self.num_tasks, device=h_base.device)
        for k in range(self.num_tasks):
            dist = self.hyperbolic_dist(z_route, self.centroids[k:k+1])
            u[:, k] = -dist.squeeze(-1)
        alpha = torch.softmax(u / self.tau, dim=-1) # [B, num_tasks]
        
        # 2. Map expert updates to Poincaré space
        v_experts = []
        for k in range(self.num_tasks):
            v_experts.append(self.exp_map(expert_updates[k])) # [B, seq_len, dim]
            
        # 3. Project to Klein coordinates and compute Lorentz factors
        w_klein_list = []
        gamma_list = []
        for k in range(self.num_tasks):
            v = v_experts[k]
            norm_sq = torch.sum(v * v, dim=-1, keepdim=True)
            w = (2.0 * v) / (1.0 + self.c * norm_sq)
            w_norm_sq = torch.sum(w * w, dim=-1, keepdim=True)
            gamma = 1.0 / torch.sqrt(torch.clamp(1.0 - self.c * w_norm_sq, min=1e-7))
            w_klein_list.append(w)
            gamma_list.append(gamma)
            
        # 4. Lorentz-weighted Einstein midpoint in Klein space
        w_merged_num = torch.zeros_like(h_base)
        w_merged_den = torch.zeros(B, h_base.shape[1], 1, device=h_base.device)
        for k in range(self.num_tasks):
            weight = alpha[:, k].view(B, 1, 1) * gamma_list[k]
            w_merged_num += weight * w_klein_list[k]
            w_merged_den += weight
            
        w_merged_klein = w_merged_num / torch.clamp(w_merged_den, min=1e-7)
        
        # Stable Klein norm clipping
        norm_klein_sq = torch.sum(w_merged_klein * w_merged_klein, dim=-1, keepdim=True)
        max_norm = 1.0 / (self.c ** 0.5) - 1e-7
        norm_klein = torch.sqrt(norm_klein_sq)
        scale_mask = (norm_klein > max_norm)
        w_merged_klein = torch.where(scale_mask, w_merged_klein * (max_norm / norm_klein), w_merged_klein)
        norm_klein_sq = torch.where(scale_mask, torch.tensor(max_norm**2, device=h_base.device), norm_klein_sq)
        
        # 5. Project Klein midpoint back to Poincaré coordinates
        denom = 1.0 + torch.sqrt(torch.clamp(1.0 - self.c * norm_klein_sq, min=1e-7))
        v_merged = w_merged_klein / denom
        
        # 6. Map back to Euclidean space and add residually
        E_merged = self.log_map(v_merged)
        return h_base + E_merged

    def hyperbolic_dist(self, x, y):
        minus_x = -x
        add_res = self.mobius_add(minus_x, y)
        norm_res = torch.norm(add_res, dim=-1, keepdim=True) + 1e-8
        val = (self.c ** 0.5) * norm_res
        val = torch.clamp(val, 0.0, 1.0 - 1e-7)
        artanh_val = 0.5 * torch.log((1.0 + val) / (1.0 - val))
        return (2.0 / (self.c ** 0.5)) * artanh_val

    def mobius_add(self, x, y):
        inner_xy = torch.sum(x * y, dim=-1, keepdim=True)
        norm_x_sq = torch.sum(x * x, dim=-1, keepdim=True)
        norm_y_sq = torch.sum(y * y, dim=-1, keepdim=True)
        num_part1 = (1.0 + 2.0 * self.c * inner_xy + self.c * norm_y_sq) * x
        num_part2 = (1.0 - self.c * norm_x_sq) * y
        numerator = num_part1 + num_part2
        denominator = 1.0 + 2.0 * self.c * inner_xy + (self.c ** 2) * norm_x_sq * norm_y_sq
        return numerator / (denominator + 1e-9)
\end{verbatim}
