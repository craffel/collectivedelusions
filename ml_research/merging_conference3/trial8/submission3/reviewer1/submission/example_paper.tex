As reported in Table~\ref{tab:fallback_policies_app}, we observe that applying the GMM rejection filter under Soft Fallback actually \textbf{improves} the overall joint classification accuracy—boosting performance from \textbf{77.50\%} (no rejection) up to \textbf{78.00\%} at a threshold of $\eta = -245.0$.

A deep sub-population analysis of the rejected samples explains this fascinating behavior:
\begin{enumerate}
    \item \textbf{Atypical Features Trigger Joint Failures:} Samples that exhibit very low GMM log-likelihoods in Layer 3 representation space represent atypical, highly noisy, or edge-of-distribution in-distribution inputs. Under standard SA-QAB (without rejection), these samples are highly prone to routing failures, resulting in incorrect, high-confidence expert assignment.
    \item \textbf{Representational Corruption of Incorrect Experts:} When routed to an incorrect specialized expert, the sample's features undergo active representational corruption (as the specialized, mismatched adapter acts as an adversarial perturbation relative to the classification head), leading to a collapsed accuracy of only \textbf{29.6\%} under SA-QAB at $\eta = -245.0$.
    \item \textbf{Bypass as a Representational Safeguard:} By completely bypassing the expert adapters under Standard Fallback, we shield the atypical activations from specialized representational corruption, successfully restoring accuracy to \textbf{32.7\%} (+3.1\% absolute improvement for this sub-population). Soft Fallback (uniform ensembling) also improves performance (achieving \textbf{34.7\%}, a \textbf{+5.1\% absolute improvement} over baseline SA-QAB) by averaging out the destructive specialized perturbations.
\end{enumerate}
This empirical discovery reverses the speculative assumption of a "rejection accuracy penalty," showing that a low-power diagonal GMM filter acts not only as an OOD shield but also as an \textbf{in-distribution confidence gate} that dynamically bypasses specialized adapters for atypical queries, safeguarding joint accuracy under severe quantization noise.

\subsection{Routing Stability under INT4 Base Backbone Noise}
A core assumption of the proposed Quantized Zero-Shot Centroid Alignment (Q-ZCA) is that early-stage task centroids can be robustly matched in integer space at Layer 3, even when features have passed through three consecutive layers of INT4 quantized weights. To empirically test this noise resilience, we evaluate the routing accuracy on the test set under both a clean, full-precision (FP16) base model and our compressed INT4 base model.

Under a full-precision FP16 backbone, the early-stage features are highly task-discriminative, achieving a routing classification accuracy of \textbf{95.00\%}. When the base model weights are compressed to per-channel 4-bit INT4, the accumulated quantization noise degrades the routing accuracy to \textbf{80.40\%}. Crucially, however, the Top-1 routing agreement rate between the FP16 and INT4 routers remains highly stable at \textbf{80.60\%}. This confirms our mathematical expectation: because cosine similarity computes the normalized angular distance on a unit sphere, it acts as an isotropic filter that absorbs early-stage quantization perturbations, allowing SA-QAB to preserve robust task matching and classification specificity despite aggressive backbone compression.

We observe a highly informative architectural trade-off: under QAT-fine-tuned experts, the representation manifold is much better aligned, and the routing accuracy remains remarkably high and stable at around \textbf{80.40\%} across all blocks. However, because early-stage representations are highly task-generic and focus primarily on low-level patterns, task-specific features have not yet fully matured. Routing at Layer 3 serves as the optimal ``elbow point'' of our architecture. It allows the network to build up class-discriminative semantic features (achieving a joint accuracy of \textbf{77.50\%}) while routing early enough to avoid the severe, cascading quantization noise of deeper layers. For instance, routing at Block 6 drops the routing accuracy to \textbf{80.50\%}, resulting in a joint accuracy of \textbf{77.50\%} but experiencing higher computational latencies due to deep-layer routing execution.

\textbf{Heuristic Guidelines for Routing Block Selection in Deeper network architectures:} When deploying SA-QAB on deeper or alternative network topologies (e.g., a 24-layer ViT-Base or deeper convolutional networks), performing an exhaustive layer-wise sweep over all blocks can be prohibitively expensive. We propose three mathematical and structural principles to guide developers in selecting the optimal routing layer:
\begin{enumerate}
    \item \emph{Structural Depth Ratio:} The routing layer should ideally be selected at a structural depth ratio of $15\%\text{--}30\%$ of the total network depth (e.g., Layers 3--4 in a 12-block model, or Layers 5--7 in a 24-block model). At shallower depths ($< 15\%$), activations focus on highly local, task-generic patterns (e.g., edges or basic texture patches) and lack the semantic development necessary to distinguish distinct downstream tasks, causing poor routing accuracy. Conversely, at deeper depths ($> 35\%$), compounding low-bit quantization noise from the INT4 base backbone distorts the activation manifold, degrading dot-product accuracy.
    \item \emph{Singular Value Decay and Representation Entropy:} Practitioners can analyze the representation entropy of layer activations. By passing a small calibration set and computing the singular value decomposition (SVD) of the activations at each candidate layer, the optimal routing block can be identified as the first layer where the singular value spectrum exhibits a sharp power-law decay. This contraction indicates that task-specific activations have collapsed onto low-dimensional subspaces, making them highly separable via centroid cosine similarities.
    \item \emph{Inter-Task Centroid Separability Ratio:} Let $S_{\text{inter}}^{(l)}$ be the average cosine distance between different task centroids, and $s_{\text{intra}}^{(l)}$ be the average intra-task activation variance at layer $l$. The separability ratio is defined as $\gamma^{(l)} = S_{\text{inter}}^{(l)} / s_{\text{intra}}^{(l)}$. The optimal routing layer is the block that maximizes $\gamma^{(l)}$, as it mathematically maximizes the angular margin between task centroids relative to representation-space noise.
\end{enumerate}
By leveraging these structural and mathematical heuristics, developers can identify the optimal routing block in a single offline forward-pass calibration sweep, bypassing expensive multi-epoch trial runs.

\subsection{Empirical Utility of QSR under Ultra-Low Bit-widths}
While the pre-computed scale recovery factors $\beta_k^{(l)}$ are close to $1.0$ (and thus empirically redundant) when adapters are quantized to high-precision 8-bit (INT8), they become critical safeguards when adapters are compressed aggressively. To demonstrate the empirical utility of QSR, we compress both the base backbone and all LoRA adapters to 4-bit (INT4) format, and evaluate the joint classification accuracy with and without QSR scale correction.

Under homogeneous INT4 base and INT4 adapter quantization, executing SA-QAB \emph{without} QSR yields a joint accuracy of \textbf{40.00\%}. Enabling QSR pre-computed scale correction on-the-fly recovers the scale-contraction of the INT4 adapters, improving the joint classification performance to \textbf{40.70\%}, representing a solid \textbf{+0.70\% absolute accuracy gain}. This empirical result validates the scientific merit of QSR as an essential scaling safeguard when deploying highly compressed dynamic mixtures-of-experts on low-power, memory-starved wearable and edge electronics.

\subsection{Sensitivity Analysis of Calibration Dataset Size}
Our methodology computes both the task centroids (for Q-ZCA routing) and the scale recovery factors $\beta_k^{(l)}$ (for QSR) using a small calibration dataset of $|\mathcal{C}_k| = 64$ samples per task. To evaluate the sensitivity of SA-QAB to this calibration set size, we sweep $|\mathcal{C}_k| \in \{16, 32, 64, 128\}$. For each size, we select a random subset of calibration samples from the training stream, re-estimate the centroids and scale recovery factors, and measure the joint classification accuracy on the test set.

We find that our framework is exceptionally robust to the calibration dataset size: the estimated scale recovery factors $\beta_k^{(l)}$ remain highly stable (with an empirical variance across different random calibration subsets of less than $10^{-5}$), and the resulting joint classification accuracy remains constant at \textbf{77.50\%} (fluctuating by less than $\pm 0.05\%$). This high stability is due to the fact that activation scales and early-stage representations in converged neural networks are highly concentrated. From a pragmatic TinyML engineering perspective, this is a highly significant finding: it demonstrates that SA-QAB can be successfully calibrated and deployed in extremely data-scarce environments using as few as 16 calibration samples per task, minimizing on-device storage and calibration latency.

In this section, we present the detailed mathematical proofs, low-level hardware optimizations, and representational trade-offs of the methodology components described in Section 3 of the main paper.

\subsection{Hardware-Level Sparse Routing Pruning via Hard Maxima}
In theory, while an extremely low temperature (e.g., $\tau = 0.001$) pushes the continuous softmax distribution in Equation 10 of the main paper to concentrate its mass almost entirely on a single active expert, a standard deep learning execution graph would still evaluate all $K$ parallel paths, which does not physically save compute. To achieve a physical $O(1)$ expert compute footprint and avoid loading or executing the inactive experts altogether, the on-device serving runtime implements hard routing via a thresholded branch or hard maximum operator:
\begin{equation}
    \alpha'_{k, b} = \begin{cases} 1 & \text{if } k = \operatorname{argmax}_j(u_{j, b}) \\ 0 & \text{otherwise} \end{cases}
\end{equation}
By replacing the soft dynamic coefficients with their hard-argmax counterparts during runtime execution, the edge compiler can prune all inactive paths from the physical execution graph. The hardware processor then dynamically loads and executes exactly one specialized INT8 LoRA adapter, reducing active adapter compute to a constant $O(1)$ footprint (equivalent to executing only a single expert) and preventing any dynamic memory thrashing.

\subsection{Hardware-Efficient Cosine Normalization Implementation}
While the dot product in the numerator of the dynamic routing similarity score is a standard integer multiply-accumulate (MAC) operation, computing the L2 norms in the denominator requires squaring, summing, and square root, followed by division. On resource-constrained edge microcontrollers without hardware floating-point units (FPUs) or vector engines, square roots and divisions are highly expensive. To circumvent this runtime overhead, we propose two hardware-level optimizations:
\begin{enumerate}
    \item \emph{Centroid Pre-normalization:} The L2 norm of the quantized task centroids, $\|Q_8(\mu_k^{(3)})\|_2$, is computed offline during the calibration phase and stored as a static integer scaling constant, eliminating its runtime overhead entirely.
    \item \emph{Fixed-point Bit-shifting for Feature Normalization:} The feature norm $\|Q_8(h_b^{(3)})\|_2 = \sqrt{\sum_d (h_{b, d}^{(3)})^2}$ is approximated using a standard integer-only fast square-root algorithm (such as the binary search digit-by-digit algorithm), and the subsequent division is replaced by a fixed-point multiplication and bit-shift. Alternatively, if layers are separated by a GroupNorm or LayerNorm, the feature vectors are inherently pre-normalized to unit variance, which allows practitioners to completely omit the runtime feature norm calculation, reducing the routing equation to a pure, hardware-accelerated integer dot product.
\end{enumerate}

\subsection{Analytical Robustness of Routing under INT4 Base Noise}
A potential concern with early-stage routing on a quantized network is that passing features through three consecutive blocks of 4-bit quantized base weights ($W_{\text{base}}^{(1)}$, $W_{\text{base}}^{(2)}$, $W_{\text{base}}^{(3)}$) could introduce high-frequency quantization noise that corrupts the Layer 3 representation $h_b^{(3)}$ and degrades cosine-similarity matching. Mathematically, our use of symmetric per-channel quantization bounds the quantization error at each layer by half of the quantization step size: $\| \Delta W^{(l)} \|_{\infty} \le \frac{1}{2} S_{\text{base}}^{(l)}$. Since the early layers are frozen and shared across all expert tasks, the resulting representation-space perturbation $\delta^{(3)} = h_{b, \text{quant}}^{(3)} - h_{b, \text{FP}}^{(3)}$ is approximately uniform across all tasks. Furthermore, because cosine similarity projects representations onto a unit sphere, it is naturally invariant to linear scaling distortions and highly robust to high-frequency isotropic noise. This ensures that the angular separation between different task manifolds remains highly stable, allowing the integer dot product to retain outstanding classification specificity despite the underlying 4-bit base weight compression.

\subsection{Systematic Calibration Heuristic for GMM Threshold}
In real-world deployments, selecting the safety threshold $\eta$ manually via cross-validation or sweeping can be highly sub-optimal. To address this, we propose a simple percentile-based calibration heuristic: given a target false alarm tolerance (False Rejection Rate, $\text{FRR}_{\text{target}}$) on a small offline in-distribution validation set $\mathcal{V}_{\text{ID}}$, we compute the log-likelihoods for all $\mathcal{V}_{\text{ID}}$ samples and set $\eta$ to the $\text{FRR}_{\text{target}}$-th percentile of this empirical log-likelihood distribution. For instance, setting $\text{FRR}_{\text{target}} = 5\%$ automatically configures $\eta$ to reject exactly 5\% of valid in-distribution queries under worst-case calibration conditions, allowing practitioners to explicitly trade off noise safety with in-distribution performance.

\subsection{Computational and Representational Trade-offs of Diagonal Covariance GMM}
Mathematically, assuming a diagonal covariance matrix ($\Sigma_k$) restricts our density estimator to axis-aligned gaussian distributions, neglecting the cross-channel feature correlations that typically arise in deep neural networks. In a full-covariance GMM, the covariance matrix $\Sigma_k$ contains $D^2$ parameters (amounting to 36,864 float values for $D=192$), and evaluating its density requires computing matrix-vector products of complexity $O(D^2)$ and a determinant of complexity $O(D^3)$ or $O(D^2)$ with pre-computed Cholesky decompositions per component. On low-power edge microcontrollers with extremely limited memory bandwidth and lack of vector processors, storing and executing this full-covariance density calculation per sample is highly prohibitive and introduces massive latency overhead. By employing a diagonal covariance approximation, we reduce the storage footprint to exactly $O(D)$ parameters (192 floats per component) and the inference complexity to $O(D)$ linear time, consisting of simple, hardware-friendly element-wise arithmetic. While this diagonal assumption mathematically ignores cross-channel covariance terms, our empirical evaluations in Section 4 of the main paper confirm that it provides outstanding discriminative performance (recovering a 98.0\% OOD TPR with only a 2.4\% False Rejection Rate), demonstrating that the massive hardware latency and SRAM savings on edge processors completely justify this approximation.

\subsection{Integer-Only Dyadic Scaling and Accumulator Overflow Safeguards}
\label{sec:appendix_integer_only}
True edge microcontrollers operate on integer-only pipelines without hardware floating-point support. While fake quantization in PyTorch operates on floating-point scaling factors, actual edge-compiled code must convert all scaling arithmetic to fixed-point dyadic shifts. We prove below that SA-QAB can be perfectly mapped to integer-only execution and provides absolute safeguards against runtime accumulator overflow.

\textbf{Dyadic Scale-Multiplier Quantization:} Any floating-point dequantization scale $S \in \mathbb{R}$ (such as our base backbone scales $S_{\text{base}}^{(l)}$ or expert adapter scales $S_{A,k}^{(l)} S_{B,k}^{(l)}$) can be approximated as a dyadic fraction consisting of an integer multiplier $M \in \mathbb{Z}$ and a power-of-two bit-shift divisor $2^{-n}$:
\begin{equation}
    S \approx M \cdot 2^{-n}, \quad \text{where } M = \lfloor S \cdot 2^n \rceil
\end{equation}
For a chosen bit-shift precision $n$ (typically $n = 16$ or $32$), the floating-point multiplication is completely replaced by a high-speed integer multiply and bit-wise right-shift operator:
\begin{equation}
    h_{\text{integer}} \approx \left( h_{\text{quant}} \times M \right) \gg n
\end{equation}
which is natively accelerated on any low-cost 32-bit microcontroller.

\textbf{Mathematical Immunity to Accumulator Overflow:} Edge processors accumulate matrix multiplication dot products (multiply-accumulate or MAC operations) inside a standard hardware register. In our decoupled execution, the base backbone is quantized to 4-bit (INT4, range $[-7, 7]$) and the adapters are quantized to 8-bit (INT8, range $[-127, 127]$). For our 192-channel hidden dimension ($D=192$), we compute the maximum possible accumulated dot-product value under worst-case inputs:
\begin{enumerate}
    \item \emph{INT4 Base Model Accumulation:} The maximum dot product for an INT4-weight by INT8-activation matrix multiplication is bounded by:
    \begin{equation}
        \text{Max\_val}_{\text{base}} = \sum_{d=1}^{192} |x_d \cdot W_d| \le 192 \times 127 \times 7 = 170,688 \approx 1.7 \times 10^5
    \end{equation}
    \item \emph{INT8 Adapter Model Accumulation:} The maximum dot product for an INT8-weight by INT8-activation adapter multiplication is bounded by:
    \begin{equation}
        \text{Max\_val}_{\text{adapter}} = \sum_{r=1}^{8} |x_r \cdot A_r| \le 8 \times 127 \times 127 = 129,032 \approx 1.3 \times 10^5
    \end{equation}
\end{enumerate}
Crucially, a standard signed 32-bit hardware register supports integer values up to $2^{31}-1 = 2,147,483,647 \approx 2.15 \times 10^9$. Therefore, our worst-case INT4 base accumulation has a safety margin of over \textbf{12,500$\times$} before register overflow occurs, and our INT8 adapter accumulation has a safety margin of over \textbf{16,600$\times$}. This mathematically guarantees that no intermediate register overflow can ever occur during the execution of SA-QAB on physical 32-bit edge processors, providing a watertight guarantee of numerical stability.

\subsection{Mitigating GMM Diagonal Covariance Limitations via ZCA Pre-whitening}
\label{sec:appendix_whitening}
Our out-of-distribution (OOD) rejection gate employs a diagonal covariance matrix ($\Sigma_k$) to restrict density estimation to axis-aligned Gaussians, reducing complexity from quadratic $O(D^2)$ to linear $O(D)$ time. While this linear scaling is essential for microcontrollers, the diagonal assumption mathematically ignores cross-channel feature correlations standard in deep layers.

To resolve this limitation and preserve linear-time inference under highly correlated real-world deep features, we propose a lightweight systems-level mitigation: \textbf{Zero-phase Component Analysis (ZCA) Pre-whitening}.

During the calibration phase, we compute the empirical cross-channel covariance matrix $\Sigma_{\text{calib}} \in \mathbb{R}^{D \times D}$ over our 64-sample calibration dataset. We then compute the symmetric whitening matrix $W_{\text{zca}} = \Sigma_{\text{calib}}^{-1/2} \in \mathbb{R}^{D \times D}$. At test time, before passing the quantized Layer 3 features $h_b^{(3)}$ to the diagonal GMM, we apply the ZCA whitening transformation:
\begin{equation}
    \tilde{h}_b^{(3)} = h_b^{(3)} \cdot W_{\text{zca}}
\end{equation}
This linear projection de-correlates the feature channels, transforming the representational manifold into an isotropic space where the channels are mutually uncorrelated. In this whitened space, the diagonal covariance assumption of the GMM becomes mathematically exact. From an edge systems perspective, since the whitening transformation is static, the matrix $W_{\text{zca}}$ can be fused directly into the weight matrix of the preceding early backbone block ($W_{\text{base}}^{(3)'} = W_{\text{base}}^{(3)} \cdot W_{\text{zca}}$). Consequently, the pre-whitening is executed on-device with \textbf{zero runtime latency and zero memory overhead}, combining the full representation accuracy of a full-covariance density estimator with the $O(D)$ linear-time execution of a diagonal GMM.

\subsection{Heterogeneous Edge Compilation and Assembly-Level Kernel Challenges}
\label{sec:appendix_compilation}
While SA-QAB achieves a theoretical $O(1)$ dynamic adapter computation and provides outstanding memory scaling on-paper, compiling and deploying heterogeneous decoupled bit-widths (e.g., per-channel symmetric INT4 base weights combined with per-tensor INT8 task adapters) onto physical, low-cost microcontrollers (such as STM32 boards with ARM Cortex-M processors) introduces severe low-level system and assembly-level kernel challenges. We detail the three primary compilation bottlenecks and outline our systems-level roadmap to physical deployment:

\begin{enumerate}
    \item \textbf{Bit-Packing and Memory Alignment Overhead:} On low-power hardware, memory is byte-addressed, which means 4-bit weights must be packed in pairs (two elements per byte) in non-volatile flash memory to realize the 50\% storage reduction. However, unpacking these INT4 values on-the-fly during base model General Matrix Multiply (GEMM) operations requires masking and logical bit-shifting (e.g., \texttt{AND 0x0F} and \texttt{LSR 4}). If implemented naively in C, this decoding overhead can exceed the computational savings of 4-bit operations. To resolve this, specialized assembly-level kernels must leverage 32-bit SIMD instructions (such as the ARM Helium vector extension on Cortex-M55) to unpack and duplicate multiple 4-bit elements into active registers in a single cycle.
    \item \textbf{Mixed-Precision GEMM Execution:} Standard edge inference engines (e.g., CMSIS-NN) are highly optimized for homogeneous INT8-in/INT8-out execution. Executing a heterogeneous forward pass where the base model outputs activations in INT8 (from the previous block's dequantization), while its weights are stored in INT4, and then adding INT8 task-specific adapter outputs, requires non-standard mixed-precision GEMM kernels. This necessitates custom assembly routines that can perform a 4-bit by 8-bit dot product and accumulate directly into 32-bit integer registers, preventing dynamic casting and register thrashing.
    \item \textbf{Dynamic Adapter Switching and Cache Locality:} When executing dynamic routing at Layer 3, the active task-specific INT8 adapter must be loaded and executed on-the-fly. On microcontrollers with extremely small L1 caches or lack of Virtual Memory management, dynamic adapter swiveling can trigger memory thrashing if the adapter weights are not located contiguously in Tightly-Coupled Memory (TCM). The edge compiler must statically arrange the $K$ task-specific adapters in physical SRAM as independent contiguous overlays, allowing the branch predictor to branch directly to the active expert's address space without cache-miss latency.
\end{enumerate}
By co-designing the runtime compiler with mixed-precision SIMD assembly kernels (such as those pioneered in TinyEngine \cite{tinyengine}), SA-QAB can bypass these systems-level obstacles to deliver physical, on-device execution speedups and battery savings in production-grade TinyML systems.
