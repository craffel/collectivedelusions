\section{Minimalist Algorithmic Implementation}
\label{app:code}

To demonstrate the simplicity and ease of integration of \textbf{Sparse Task Arithmetic (STA)}, we provide a complete, self-contained PyTorch implementation in Listing~\ref{lst:pytorch_sta}. Unlike other sparse model merging methods that require thousands of lines of coordinate sign-checking and voting systems, STA is implemented in just three core lines of logic.

\begin{figure*}[h]
\begin{minipage}{\textwidth}
\begin{verbatim}
import torch

def sparse_task_arithmetic(base_model, task_experts, survival_density=20.0, lambda_coeff=0.3):
    """
    Apply Sparse Task Arithmetic (STA) to merge multiple task-specific expert models.
    
    Args:
        base_model (dict): State dict of the pre-trained base model.
        task_experts (list of dicts): List of state dicts of task-specific expert models.
        survival_density (float): Percentage of weights to retain per layer (s in %).
        lambda_coeff (float): Scaling factor for the merged updates.
    """
    merged_state_dict = {}
    
    for key in base_model.keys():
        # Only merge floating point parameter tensors (weights/biases)
        if not base_model[key].is_floating_point():
            merged_state_dict[key] = base_model[key]
            continue
            
        # 1. Compute task vectors: v_k = \theta_k - \theta_0
        task_vectors = [expert[key] - base_model[key] for expert in task_experts]
        
        # 2. Uniform layer-wise magnitude pruning
        sparse_updates = []
        for v in task_vectors:
            threshold = torch.quantile(v.abs(), 1.0 - (survival_density / 100.0))
            mask = (v.abs() >= threshold).float()
            sparse_updates.append(v * mask)
            
        # 3. Direct linear summation (No sign resolution)
        merged_state_dict[key] = base_model[key] + lambda_coeff * sum(sparse_updates)
        
    return merged_state_dict
\end{verbatim}

\section{Implementation Details and Hyperparameters}
All task checkpoints (MNIST, FashionMNIST, CIFAR-10, SVHN) were evaluated on a Vision Transformer (ViT-B-32) architecture fine-tuned from pre-trained CLIP weights. The linear heads for each dataset are loaded onto the unified representation of the backbone.
We sweep the survival density $s \in \{5\%, 10\%, 20\%, 50\%\}$ using layer-wise absolute thresholding. The scaling coefficient $\lambda$ was set to $0.3$ across all baselines and methods to ensure an unbiased, parameter-free comparison. No additional learning rate or test-time optimization was applied.
All measurements are evaluated using standard PyTorch metrics on a subset of 2,048 validation samples per dataset to maintain high statistical significance and guarantee physical reproducibility.
