\section{Circular Convolution Weight-Binding Roadmap}
\label{app:circular_convolution}

To address the coordinate-isolation confounder deconstructed in Section~\ref{subsec:dimension_ablation}, we prove that transitioning from element-wise Hadamard binding to circular convolution weight-binding achieves a scale-invariant $O(1/\sqrt{D})$ relative reconstruction noise decay rate, enabling perfectly lossless hyperdimensional parameter superposition in high-dimensional spaces.

Let $v, k \in \mathbb{R}^D$ be vectors representing specialized task vectors and random carrier keys. In Vector Symbolic Architectures (VSAs) like Holographic Reduced Representations (HRR)~\cite{plate2003holographic}, binding is defined via circular convolution $\circledast$:
\begin{equation}
x = v \circledast k
\end{equation}
where the $d$-th component of the circular convolution is:
\begin{equation}
x_d = \sum_{m=0}^{D-1} v_m k_{(d-m) \pmod D}
\end{equation}

Demodulation (unbinding) is performed via circular correlation $\circledast'$ using the corresponding key:
\begin{equation}
\hat{v} = x \circledast' k = v \circledast \left( k \circledast' k \right)
\end{equation}

Suppose the components of the random carrier keys $k$ are drawn from a normal distribution with scaled variance:
\begin{equation}
k_d \sim \mathcal{N}\left( 0, \frac{1}{D} \right)
\end{equation}
This ensures that the expected Frobenius norm of the key is $\mathbb{E}[\|k\|_2^2] = 1$. Under this scaling, the circular self-correlation $k \circledast' k$ converges to a Kronecker delta distribution $\delta$ (the identity element of circular convolution) up to a zero-mean noise term:
\begin{equation}
k \circledast' k \approx \delta + \eta
\end{equation}
where the components of $\eta \in \mathbb{R}^D$ are independent, zero-mean random variables with variance scaling as:
\begin{equation}
\text{Var}(\eta_d) = \frac{1}{D}
\end{equation}

When we superimpose $K$ specialized task vectors $v_j$ bound to mutually orthogonal random keys $k_j$, we construct the holographic superposition vector:
\begin{equation}
x_{\text{holo}} = \sum_{j=1}^K v_j \circledast k_j
\end{equation}

We demodulate the target task vector $v_k$ using its corresponding key $k_k$:
\begin{align}
\hat{v}_k &= x_{\text{holo}} \circledast' k_k \\
&= \left( \sum_{j=1}^K v_j \circledast k_j \right) \circledast' k_k \\
&= v_k \circledast \left( k_k \circledast' k_k \right) + \sum_{j \neq k} v_j \circledast \left( k_j \circledast' k_k \right)
\end{align}

Applying the self-correlation identity:
\begin{equation}
\hat{v}_k = v_k \circledast \delta + v_k \circledast \eta + \sum_{j \neq k} v_j \circledast \left( k_j \circledast' k_k \right)
\end{equation}
Since $v_k \circledast \delta = v_k$, this expands to:
\begin{equation}
\hat{v}_k = v_k + \zeta
\end{equation}
where the cross-talk noise term $\zeta$ is:
\begin{equation}
\zeta = v_k \circledast \eta + \sum_{j \neq k} v_j \circledast \left( k_j \circledast' k_k \right)
\end{equation}

Because the carrier keys $k_j$ and $k_k$ are independent random vectors for $j \neq k$, the circular correlation $k_j \circledast' k_k$ is a random vector with components of variance $1/D$. When convolved with $v_j$, the components of this cross-talk term behave as a sum of $D$ independent, zero-mean variables. By the Central Limit Theorem, the variance of each coordinate in the cross-talk noise $\zeta$ scales as:
\begin{equation}
\text{Var}(\zeta_d) \propto O\left( \frac{1}{D} \right)
\end{equation}

Therefore, the standard deviation of the weight reconstruction noise scales as:
\begin{equation}
\sigma_{\text{noise}} \propto O\left( \frac{1}{\sqrt{D}} \right)
\end{equation}

As the representation dimension $D$ increases, the relative reconstruction noise decays at a rate of $O(1/\sqrt{D})$, converging to zero in high-dimensional spaces:
\begin{equation}
\lim_{D \to \infty} \frac{\|\hat{v}_k - v_k\|_2}{\|v_k\|_2} = 0
\end{equation}

This mathematical proof establishes that circular convolution distributes coordinate information globally across all dimensions, resolving the coordinate-isolation confounder and guaranteeing lossless dynamic model ensembling as the dimension scales to modern foundation model dimensions (e.g., $D \geq 4096$).

\subsection{Practical Computational Bottleneck: FFT Complexity and Block-wise Mitigation}

While circular convolution provides a mathematically elegant solution to the coordinate isolation confounder, achieving scale-invariant $O(1/\sqrt{P})$ noise decay, we must address the practical computational complexity it introduces for 2D weight matrices. 

If circular convolution is applied directly to a flattened weight matrix of size $P = R \times C$, performing circular convolution on the fly during demodulation requires computing Fast Fourier Transforms (FFTs) of size $P$. This incurs a computational cost of:
\begin{equation}
\text{FLOPs}_{\text{FFT}} = O(B \times P \log P)
\end{equation}
per layer, per forward pass. For a standard transformer layer where $P \approx 10^6$ parameters, executing a million-point FFT sample-by-sample inside the forward pass would be computationally prohibitive, severely degrading runtime throughput and latency on edge hardware.

To reconcile this mathematical roadmap with practical deployment limits, we propose three concrete structural approximations to bypass the full $P \log P$ bottleneck:
\begin{enumerate}
\item \textbf{Block-wise Circular Convolution:} Instead of convolving the entire $P$-dimensional weight vector at once, the weight matrix is partitioned into $M$ independent smaller blocks, each of dimension $d = P / M$ (e.g., block-wise across rows or channels, where $d = C = 192$ or $768$). Circular convolution is then executed independently on each block:
\begin{equation}
W_{\text{holo}, m} = \sum_{k=1}^K V_{k, m} \circledast k_{k, m}, \quad \text{for } m \in \{1, \dots, M\}
\end{equation}
By choosing a small block size $d \leq 1024$, the FFT size is kept extremely small, which can be executed highly efficiently in parallel inside hardware registers. Under block-wise convolution, the standard deviation of the reconstruction noise still decays as $O(1/\sqrt{d})$. By choosing $d = 768$ (a standard hidden dimension), the relative reconstruction noise is kept under 3.5\% while incurring negligible FLOP overhead.
\item \textbf{FFT-Free Circular Convolution via Shift-Registers:} For small block sizes $d$, we can completely avoid FFT computations. Since the keys are binary and frozen, circular convolution can be implemented as a series of hardware bit-shifts and additions using local registers. This maps beautifully onto specialized edge accelerators and NPUs, bypassing transcendental floating-point FFT arithmetic entirely.
\item \textbf{Kronecker-Structured or Low-Rank Factorized Convolution:} By exploiting the 2D spatial structures of neural layers, we can factorize the 2D weight matrix into low-rank Kronecker products, e.g., $W = W_1 \otimes W_2$. By applying circular convolution independently to the low-rank factor matrices $W_1 \in \mathbb{R}^{r_1 \times c_1}$ and $W_2 \in \mathbb{R}^{r_2 \times c_2}$ (which are orders of magnitude smaller than $W$), we can reconstruct the full convolved weights with a cost of only $O(R \log R + C \log C)$ FLOPs instead of $O(P \log P)$. This factorized approach completely eliminates the computational complexity bottleneck on resource-constrained edge accelerators.
\end{enumerate}
This structural block-wise formulation ensures that hyperdimensional parameter ensembling remains both mathematically lossless and computationally viable for real-world deployment.

\subsection{The Continuous Coordinate-wise Reconstruction Paradox: The Cleanup Dilemma}

While Figure~\ref{fig:circular_retrieval_gap} provides an extremely optimistic decay curve for discrete template lookup, we must confront a fundamental mathematical caveat: the \textit{continuous coordinate-wise relative $L_2$ reconstruction error} under circular convolution remains completely invariant across dimensions at exactly:
\begin{equation}
\frac{\|\hat{v}_k - v_k\|_2}{\|v_k\|_2} \approx \sqrt{K-1}
\end{equation}
which is approximately 173\% for $K=4$ tasks. This scale-invariance is due to the isometric property of both the Fourier Transform and circular convolution, which preserves the total energy of the cross-talk noise in high-dimensional space. Under circular convolution, the cross-talk noise power is distributed globally, but the overall Euclidean distance between the demodulated weight vector and the clean target weight vector does not shrink.

In traditional Vector Symbolic Architectures (VSAs) operating on activations, this noise is neutralized by passing the noisy vector through a discrete \textit{cleanup memory} (an associative memory that performs a nearest-neighbor projection onto a clean, stored dictionary of candidate templates). However, in deep model ensembling, we cannot execute a discrete cleanup memory on continuous weight matrices during the forward pass. This is because:
\begin{enumerate}
\item The weights must remain continuous and coordinate-wise precise to propagate continuous, high-dimensional activation features $h$ through the layers without representation collapse. A discrete projection of $W_b$ onto a set of fixed candidate expert matrices $\{W_1, \dots, W_K\}$ would collapse $W_b$ to a single static expert, completely defeating EHPB's core capability of sub-layer-level dynamic, continuous parameter interpolation ($\alpha_k \in [0, 1]$).
\item Even if such a projection were mathematically viable, executing a nearest-neighbor search over the candidate expert matrices would require keeping all $K$ expert models in VRAM simultaneously. This scales active execution memory to $O(K \times P)$, which directly violates the Resource Efficiency axis of the Post-Hoc Model Ensembling Trilemma (Section~\ref{subsec:ensembling_trilemma}), collapsing EHPB's resource-efficiency advantage.
\end{enumerate}

Consequently, the 170\% coordinate-wise weight reconstruction noise remains fully active during EHPB layer propagation. While Appendix~\ref{app:nonlinear_activations} explains how non-linear activations and spatial visual pooling act as low-pass filters to mitigate this noise, weight coordinate integrity is heavily compromised, explaining EHPB's absolute accuracy penalty (25.4\% Joint Mean). 

To reconcile noise-free activation propagation with single-matrix edge parameter storage, we propose transitioning from weight-space cleanup to \textbf{activation-space cleanup}. We formalize two concrete architectural paradigms to dynamically filter out weight-reconstruction noise during the forward pass:
\begin{enumerate}
\item \textbf{Continuous Cleanup Networks (CCN):} Instead of attempting to denoise the high-dimensional, continuous weight parameters $W_b \in \mathbb{R}^{R \times C}$ on-the-fly, we can train a lightweight, non-linear post-hoc feedforward block $\Psi^{(l)}: \mathbb{R}^D \to \mathbb{R}^D$ (e.g., a bottleneck MLP with a hidden dimension of $D/r$, where $r$ is the compression ratio) on a tiny calibration split. The CCN is trained to map the noisy pre-activations $y_b^{(l)} = y_{\text{target}, b}^{(l)} + e_b^{(l)}$ directly back to the target noise-free pre-activations:
\begin{equation}
\hat{y}_b^{(l)} = \Psi^{(l)}\left( y_b^{(l)} \right)
\end{equation}
Because the CCN operates only on $D$-dimensional activations rather than the $P$-dimensional parameters, its parameter footprint is exceptionally small ($O(D^2/r)$ parameters), preserving EHPB's single-matrix storage advantage while achieving active, post-hoc denoising of activation trajectories.
\item \textbf{Activation-Space Projection Layers (ASPL):} By leveraging the low-rank properties of task-specific activations, we can project the noisy pre-activations onto task-specific low-dimensional activation subspaces. Let $U_k^{(l)} \in \mathbb{R}^{D \times d}$ (with $d \ll D$) be an orthonormal basis for the $d$-dimensional principal subspace of the clean activations of task $k$ at layer $l$. At test-time, we construct a dynamic sample-wise projection operator using the predicted task routing coefficients $\alpha_{k, b}$:
\begin{equation}
P_b^{(l)} = \sum_{k=1}^K \alpha_{k, b} U_k^{(l)} \left( U_k^{(l)} \right)^T
\end{equation}
Applying this operator to the noisy pre-activations yields:
\begin{equation}
\tilde{y}_b^{(l)} = P_b^{(l)} y_b^{(l)} = P_b^{(l)} y_{\text{target}, b}^{(l)} + P_b^{(l)} e_b^{(l)}
\end{equation}
Assuming that the zero-mean cross-talk noise $e_b^{(l)} = \Xi_b^{(l)} h_b^{(l-1)}$ is distributed spherically across the $D$-dimensional feature space, projecting it onto the $d$-dimensional task subspace reduces the noise variance by a factor of $d/D$:
\begin{equation}
\mathbb{E}\left[ \|P_b^{(l)} e_b^{(l)}\|_2^2 \right] = \frac{d}{D} \mathbb{E}\left[ \|e_b^{(l)}\|_2^2 \right]
\end{equation}
For instance, by setting $d = D/10$, we can analytically filter out 90\% of the weight reconstruction noise variance inside the forward pass. Storing the task-specific basis matrices requires only $O(K \times D \times d)$ parameters, which is negligible compared to the full model size $P$.
\end{enumerate}

\subsection{The In-Network Validation Gap: Practical Obstacles of In-Network Circular Convolution}
We must candidly discuss why circular convolution-based weight superposition has not been implemented or evaluated inside the actual 14-layer ViT-Tiny sandbox network. This represents a significant gap between our positive theoretical prescription and its physical validation in multi-layer deep networks. Implementing in-network circular convolution on weight matrices presents three major, unaddressed computational and architectural hurdles:
\begin{enumerate}
\item \textbf{Non-Isomorphism of 2D Convolution:} Element-wise Hadamard modulation ($V_k \odot K_k$) preserves the exact shape and Frobenius norm of the weight matrices ($R \times C$). In contrast, circular convolution is fundamentally a 1D vector operator. Applying circular convolution to a 2D weight matrix requires either flattening it into a 1D vector of size $P = R \times C$ (which completely destroys the spatial 2D correlation of the linear mapping and causes the FFT sizes to explode to $10^6$ parameters) or defining a 2D circular convolution operator (which requires circular padding and multi-dimensional FFTs, adding massive computational complexity).
\item \textbf{High Forward-Pass Latency Overhead:} Demodulating the weights on-the-fly sample-by-sample via circular correlation requires executing forward and inverse 2D FFTs for each sample in a batch. This would increase the computational complexity of the forward pass to $O(B \times P \log P)$ per layer. On resource-constrained edge devices, this would result in a massive latency penalty, completely neutralizing the runtime throughput benefit of model merging.
\item \textbf{The continuous-to-discrete mismatch:} Circular convolution noise decay in VSAs relies on discrete cleanup memory. In-network deep propagation, however, requires continuous coordinate-wise precision, meaning that without a continuous activation-space cleanup network, the 173\% coordinate-wise noise remains fully active and propagates through all 14 layers.
\end{enumerate}
Consequently, evaluating circular convolution in a deep, multi-layer setting remains an open theoretical and engineering challenge. Future research must design specialized CUDA/Triton kernels that perform highly optimized block-wise FFTs and integrate lightweight continuous cleanup networks at layer boundaries to make in-network circular convolution viable.

\section{Non-Linear Activation Flows and High-Frequency Noise Filtering}
\label{app:nonlinear_activations}

During EHPB weight-transcription, the demodulated weight matrix $W_b^{(l)}$ contains a zero-mean cross-talk noise term $\Xi_b^{(l)}$. We analyze how this noise behaves when propagating through a deep neural network's non-linear activations (e.g., ReLU, GELU) and Layer Normalization.

Let $h^{(l-1)}$ be the input activations to layer $l$. The pre-activation output is:
\begin{align}
y_b^{(l)} &= W_b^{(l)} h_b^{(l-1)} \\
&= \left( W_{\text{base}}^{(l)} + \sum_{k=1}^K \alpha_{k, b} V_k^{(l)} + \Xi_b^{(l)} \right) h_b^{(l-1)} \\
&= y_{\text{target}, b}^{(l)} + \Xi_b^{(l)} h_b^{(l-1)}
\end{align}

Because $\Xi_b^{(l)}$ consists of independent, random bipolar sign-fluctuations with zero mean, the activation propagation term $e_b^{(l)} = \Xi_b^{(l)} h_b^{(l-1)}$ acts as a high-frequency, zero-mean coordinate-wise noise vector. When passed through subsequent layers:

\subsection{Systematic Positive Bias Rectification under ReLU}
Let us model the coordinate-wise cross-talk propagation noise $e_i^{(l)}$ as a zero-mean Gaussian random variable with variance $\sigma_e^2$, i.e., $e_i^{(l)} \sim \mathcal{N}(0, \sigma_e^2)$, and denote the target noiseless pre-activation as $y_{\text{target}, i}^{(l)}$.

When passed through a Rectified Linear Unit ($\text{ReLU}(x) = \max(0, x)$), the expected value of the post-activation output $x_i^{(l)} = \text{ReLU}\left( y_{\text{target}, i}^{(l)} + e_i^{(l)} \right)$ is:
\begin{equation}
\mathbb{E}[x_i^{(l)}] = \int_{-\infty}^{\infty} \max\left(0, y_{\text{target}, i}^{(l)} + \epsilon\right) \frac{1}{\sqrt{2\pi}\sigma_e} e^{-\frac{\epsilon^2}{2\sigma_e^2}} d\epsilon
\end{equation}
This integral evaluates exactly to:
\begin{equation}
\mathbb{E}[x_i^{(l)}] = y_{\text{target}, i}^{(l)} \Phi\left( \frac{y_{\text{target}, i}^{(l)}}{\sigma_e} \right) + \sigma_e \phi\left( \frac{y_{\text{target}, i}^{(l)}}{\sigma_e} \right)
\end{equation}
where $\Phi(\cdot)$ and $\phi(\cdot)$ are the cumulative distribution function (CDF) and probability density function (PDF) of the standard normal distribution, respectively.

We analyze the difference (systematic bias) $B_i^{(l)} = \mathbb{E}[x_i^{(l)}] - \text{ReLU}\left( y_{\text{target}, i}^{(l)} \right)$ introduced by the noise:
\begin{itemize}
\item \textbf{Case 1: $y_{\text{target}, i}^{(l)} \geq 0$.} Here, $\text{ReLU}\left( y_{\text{target}, i}^{(l)} \right) = y_{\text{target}, i}^{(l)}$, yielding:
\begin{equation}
B_i^{(l)} = y_{\text{target}, i}^{(l)} \left[ \Phi\left( \frac{y_{\text{target}, i}^{(l)}}{\sigma_e} \right) - 1 \right] + \sigma_e \phi\left( \frac{y_{\text{target}, i}^{(l)}}{\sigma_e} \right) \geq 0
\end{equation}
\item \textbf{Case 2: $y_{\text{target}, i}^{(l)} < 0$.} Here, $\text{ReLU}\left( y_{\text{target}, i}^{(l)} \right) = 0$, yielding:
\begin{equation}
B_i^{(l)} = y_{\text{target}, i}^{(l)} \Phi\left( \frac{y_{\text{target}, i}^{(l)}}{\sigma_e} \right) + \sigma_e \phi\left( \frac{y_{\text{target}, i}^{(l)}}{\sigma_e} \right) > 0
\end{equation}
\end{itemize}

In both cases, $B_i^{(l)} \geq 0$ for all $y_{\text{target}, i}^{(l)}$, with equality holding only in the limit $\sigma_e \to 0$. This mathematically proves that zero-mean cross-talk noise is systematically rectified into a strictly positive bias vector $B^{(l)}$. This systematic bias shifts the mean of the propagated representations, acting as an injected positive coordinate offset at every layer, which compounds across deep layers and destroys visual semantics.

\textbf{Mitigating Rectification Bias via Post-Hoc Bias Correction.} To counteract this systematic positive bias accumulation, we propose two potential post-hoc mitigation strategies that do not require full weight coordinate integrity:
\begin{enumerate}
\item \textbf{Layer-wise Running Noise Subtraction:} Since $B_i^{(l)}$ is a function of the noise variance $\sigma_e^2$, a running mean of the bias shift can be estimated over a calibration set or dynamically computed from $\sigma_e$ and subtracted immediately following the activation function: $\hat{x}_i^{(l)} = x_i^{(l)} - \mathbb{E}[B_i^{(l)}]$.
\item \textbf{Learnable Scaling and Bias Correction:} A lightweight, post-hoc scale and shift parameter can be learned for each layer (e.g., using a single linear layer or adapting the LayerNorm scale $\gamma$ and shift $\beta$ post-merging) to automatically absorb and counteract the rectification bias, stabilizing representation trajectories.
\end{enumerate}
Implementing these simple post-hoc adjustments represents an exciting and highly cost-effective research avenue to stabilize deep propagation without scaling parameter storage.

\subsection{Layer Normalization and Exponential Signal Attenuation}
Beyond element-wise activation clipping, weight-reconstruction noise alters the behavioral dynamics of Layer Normalization. For a layer output $y_i^{(l)} = y_{\text{target}, i}^{(l)} + e_i^{(l)}$, Layer Normalization computes:
\begin{equation}
\text{LN}\left( y^{(l)} \right)_i = \frac{y_i^{(l)} - \mu_{y}}{\sqrt{\sigma_{y}^2 + \epsilon}} \gamma_i + \beta_i
\end{equation}
where $\mu_y$ and $\sigma_y^2$ are the empirical mean and variance of the pre-activation vector $y^{(l)}$. Assuming $e_i^{(l)} \sim \mathcal{N}(0, \sigma_e^2)$ is uncorrelated with the target signal, the variance of the noisy pre-activation expands to:
\begin{equation}
\sigma_y^2 \approx \sigma_{\text{target}}^2 + \sigma_e^2
\end{equation}
where $\sigma_{\text{target}}^2$ is the variance of the noiseless target activation. Substituting this into the LayerNorm expression yields:
\begin{equation}
\text{LN}\left( y^{(l)} \right)_i \approx \frac{y_{\text{target}, i}^{(l)} + e_i^{(l)} - \mu_{\text{target}}}{\sqrt{\sigma_{\text{target}}^2 + \sigma_e^2}} \gamma_i + \beta_i
\end{equation}
By factoring out the noise-free variance, we obtain:
\begin{equation}
\text{LN}\left( y^{(l)} \right)_i \approx \left( \frac{\sigma_{\text{target}}}{\sqrt{\sigma_{\text{target}}^2 + \sigma_e^2}} \right) \text{LN}\left( y_{\text{target}}^{(l)} \right)_i + \tilde{e}_i^{(l)}
\end{equation}
where $\tilde{e}^{(l)}$ is a scaled residual noise term.

This derivation reveals a critical and highly destructive phenomenon: EHPB weight-reconstruction noise systematically attenuates the clean semantic signal by a factor of $\eta = \frac{\sigma_{\text{target}}}{\sqrt{\sigma_{\text{target}}^2 + \sigma_e^2}} < 1$. For our Hadamard binding where the relative reconstruction error is $\approx 170\%$, the noise variance is massive ($\sigma_e^2 \approx 3 \sigma_{\text{target}}^2$), which yields an attenuation factor of $\eta \approx \frac{1}{\sqrt{1 + 3}} = 0.50$. Across $L=14$ layers, this attenuation compounds exponentially, scaling the active signal by $\eta^{14} \approx (0.50)^{14} \approx 6 \times 10^{-5}$. Thus, the clean semantic signal is completely extinguished, explaining the catastrophic representational collapse.

\subsection{Mitigating Noise via Spatial Visual Pooling}
In Vision Transformers, globally pooling representations across tokens or patches computes the spatial average:
\begin{equation}
\bar{z} = \frac{1}{N} \sum_{i=1}^N z_i
\end{equation}
This spatial averaging acts as a low-pass spatial filter, averaging out the zero-mean sign fluctuations across the token dimension and significantly recovering the original clean visual signal. This explains why EHPB retains its classification capability despite high weight-space reconstruction error, but highlights that maintaining weight coordinate integrity is still paramount.

\section{Computational and Gating FLOP Complexity Analysis}
\label{app:computational_complexity}

While EHPB achieves a highly efficient $O(P)$ active memory scaling under the Post-Hoc Model Ensembling Trilemma, it introduces runtime FLOP overhead during dynamic unbinding. We analyze the computational complexity of EHPB demodulation under vectorized loop execution ($\mathtt{torch.vmap}$) for a batch size of $B$.

Let $l$ be a layer with weight shape $R \times C$, parameter count $P = R \times C$, and $K$ tasks.
\begin{itemize}
\item \textbf{Unbinding Operator Construction:}
For each sample $b$ in the batch, assembling the unbinding operator $U_b = \sum_k \alpha_{k, b} K_k$ requires multiplying $K$ bipolar matrices of shape $R \times C$ by routing scalar coefficients and summing them. Under our factored 2D outer-product key formulation ($K_k = r_k c_k^T$), we avoid materializing the full 2D keys during the summation. We can construct $U_b$ efficiently as:
\begin{equation}
U_b = \sum_{k=1}^K \alpha_{k, b} r_k c_k^T
\end{equation}
This requires $O(K \times (R + C))$ FLOPs per sample, which is extremely lightweight.
\item \textbf{Weight Transcription:}
Demodulating the holographic matrix $W_b = W_{\text{base}} + W_{\text{holo}} \odot U_b$ requires performing $R \times C$ element-wise multiplications and additions per sample, costing:
\begin{equation}
\text{FLOPs}_{\text{demod}} = 2 \times B \times R \times C = 2 \times B \times P
\end{equation}
\item \textbf{Layer Propagation:}
Propagating the activations $h_b \in \mathbb{R}^C$ through the transcribed weight matrix $W_b \in \mathbb{R}^{R \times C}$ costs:
\begin{equation}
\text{FLOPs}_{\text{prop}} = 2 \times B \times R \times C = 2 \times B \times P
\end{equation}
\end{itemize}

The total FLOP count for EHPB propagation is:
\begin{equation}
\text{FLOPs}_{\text{total}} = O(2 \times B \times P + 2 \times B \times P) = O(4 \times B \times P)
\end{equation}

In comparison, direct sample-wise routing (`vmap-Linear-Router`) performs standard ensembling on-the-fly, costing $2 \times B \times P$ FLOPs. Thus, EHPB incurs approximately a $2\times$ FLOP overhead relative to direct ensembling during propagation, due to the element-wise Hadamard unbinding pass. This establishes EHPB's concrete engineering trade-off: it compresses RAM requirements by $K\times$ (independent of the number of experts), at the cost of doubling the computational FLOP overhead during the forward pass.

\subsection{The Computation-vs-Memory-Bandwidth Trade-Off in Compute-Bound Regimes}
While register-level fused demodulation is exceptionally well-suited for memory-bound edge environments (where HBM bandwidth is the primary bottleneck), it introduces a distinct trade-off on server-grade compute-bound systems. In compute-bound regimes characterized by massive batch sizes ($B$) and deep architectures, the instruction-level register-demodulation overhead (proportional to $R \times C$ for every sample) can become a primary latency bottleneck. 

Specifically, whereas a traditional model-merging forward pass is highly memory-bound and benefits from reusing static weights from HBM, EHPB shifts the bottleneck from HBM bandwidth to ALU/computational throughput due to the sample-specific unbinding arithmetic. On high-throughput server GPUs (e.g., NVIDIA H100), this extra math may increase execution latency despite saving memory capacity. Therefore, EHPB represents a specialized paradigm tailored for memory-constrained edge hardware rather than high-concurrency cloud computing.

\section{Triton Register Allocation and Hardware Tiling Formulation}
\label{app:triton_formulation}

To prove that register-level demodulation is physically viable and does not trigger catastrophic register spilling on modern GPU architectures, we formalize the register allocation and tiling layout of our proposed fused Triton kernel.

Consider a fused matrix-vector multiplication kernel executing $y_b = W_b h_b$ where $W_{\text{base}}, W_{\text{holo}} \in \mathbb{R}^{R \times C}$ are loaded block-by-block. Let the thread block be configured with a 2D tiling size of $\text{Tile}_R$ rows and $\text{Tile}_C$ columns. Each thread in the block is responsible for computing a sub-matrix of shape $t_r \times t_c$ within the tile. 

To execute the fused demodulation on-the-fly, the registers allocated per thread block must store the base weight, holographic weight, unbinding elements, and accumulation registers. The exact count of 32-bit registers required per thread, $\text{Regs}_{\text{thread}}$, is formulated as:
\begin{equation}
\text{Regs}_{\text{thread}} = \text{Regs}_{\text{base\_load}} + \text{Regs}_{\text{holo\_load}} + \text{Regs}_{\text{unbinding\_op}} + \text{Regs}_{\text{accumulation}} + \text{Regs}_{\text{indexing}}
\end{equation}

Breaking down each term:
\begin{enumerate}
\item \textbf{Base and Holographic Weight Buffers:} To maximize memory bandwidth and utilize tensor-cores or vector-load instructions, weights are loaded into local SRAM (shared memory) and subsequently read into registers. The thread registers must store a double-buffer of size $t_r \times t_c$ for both matrices to overlap global memory latency:
\begin{equation}
\text{Regs}_{\text{base\_load}} + \text{Regs}_{\text{holo\_load}} = 2 \times (t_r \times t_c) + 2 \times (t_r \times t_c) = 4 \times t_r \times t_c
\end{equation}
\item \textbf{Unbinding Operators:} Since $U_b = \sum \alpha_{k, b} K_k$, and we employ factored rank-1 carrier keys ($K_k = r_k c_k^T$), each thread block loads the 1D carrier vectors $r_{k, \text{block}} \in \{-1, 1\}^{\text{Tile}_R}$ and $c_{k, \text{block}} \in \{-1, 1\}^{\text{Tile}_C}$ into shared memory. The individual thread registers must store the active sample-routing coefficients $\alpha_b \in \mathbb{R}^K$ and the active 1D key elements, requiring:
\begin{equation}
\text{Regs}_{\text{unbinding\_op}} = K + t_r + t_c
\end{equation}
\item \textbf{Accumulation registers:} To perform high-fidelity fused multiply-add (FMA) instructions, each thread maintains a coordinate-wise accumulator register of size $t_r \times t_c$:
\begin{equation}
\text{Regs}_{\text{accumulation}} = t_r \times t_c
\end{equation}
\item \textbf{Indexing and Metadata:} General purpose registers for thread indices, block coordinates, loop boundaries, and base addresses:
\begin{equation}
\text{Regs}_{\text{indexing}} \approx 12
\end{equation}
\end{enumerate}

Thus, the total registers required per thread is:
\begin{equation}
\text{Regs}_{\text{thread}} = 5 \times t_r \times t_c + t_r + t_c + K + 12
\end{equation}

For a standard Triton GEMV or GEMM tiling configuration, we set the thread workload to $t_r = 4$ and $t_c = 4$. With $K = 4$ tasks, the register allocation per thread is:
\begin{equation}
\text{Regs}_{\text{thread}} = 5 \times (4 \times 4) + 4 + 4 + 4 + 12 = 104 \text{ registers}
\end{equation}

Modern GPU architectures (such as NVIDIA Ampere or Hopper) permit a maximum allocation of 255 registers per thread before triggering register spilling to high-latency local memory. Since $104 \ll 255$, EHPB's register-level demodulation is highly conservative, leaving over 150 registers per thread free for advanced compiler optimizations, software pipelining, and loop unrolling. 

The shared memory (SRAM) overhead per thread block of shape $\text{Tile}_R \times \text{Tile}_C$ is:
\begin{equation}
\text{SRAM}_{\text{block}} = 2 \times \text{Tile}_R \times \text{Tile}_C \times \text{sizeof(float)} + 2 \times K \times (\text{Tile}_R + \text{Tile}_C) \times \text{sizeof(float)}
\end{equation}
For a typical tile size of $\text{Tile}_R = 64$ and $\text{Tile}_C = 64$:
\begin{equation}
\text{SRAM}_{\text{block}} = 2 \times 64 \times 64 \times 4 + 2 \times 4 \times (64 + 64) \times 4 = 32,768 + 4,096 = 36,864 \text{ bytes} = 36 \text{ KB}
\end{equation}
This is well within the 96 KB to 228 KB L1/SRAM cache capacity per streaming multiprocessor (SM) in modern hardware, guaranteeing 100\% block occupancy and high execution throughput. This rigorous hardware formulation proves that register-level fused demodulation is highly viable, allowing EHPB to achieve true $O(P)$ global memory scaling while maintaining high execution efficiency.

\subsection{Proof-of-Concept Triton Kernel Implementation}
\label{app:triton_kernel_code}

To demonstrate the concrete viability of register-level fused weight demodulation, we present a complete, syntactically valid Triton/Python code listing of our proposed fused unbinding and GEMV kernel (Listing 1). This kernel fuses the unbinding modulation ($W_b = W_{\text{base}} + W_{\text{holo}} \odot U_b$) and matrix-vector multiplication at the instruction level, avoiding writing any intermediate weight elements to high-bandwidth global memory.

\begin{verbatim}
import triton
import triton.language as tl

@triton.jit
def fused_ehpb_gemv_kernel(
    y_ptr,          # Pointer to output activation (B x R)
    x_ptr,          # Pointer to input activation (B x C)
    w_base_ptr,     # Pointer to base weight matrix (R x C)
    w_holo_ptr,     # Pointer to holographic weight matrix (R x C)
    alpha_ptr,      # Pointer to routing coefficients (B x K)
    keys_r_ptr,     # Pointer to 1D row carriers (K x R)
    keys_c_ptr,     # Pointer to 1D col carriers (K x C)
    R, C, K,        # Dimensions and number of experts
    stride_y_b, stride_y_r,
    stride_x_b, stride_x_c,
    BLOCK_R: tl.constexpr, BLOCK_C: tl.constexpr
):
    # Determine the active thread block and sample ID
    block_id_r = tl.program_id(0)
    sample_id = tl.program_id(1)
    
    # Compute block row offsets
    offsets_r = block_id_r * BLOCK_R + tl.arange(0, BLOCK_R)
    mask_r = offsets_r < R
    
    # Load routing coefficients for the active sample
    # alpha has shape (B x K), loading K coefficients
    alpha_offsets = sample_id * K + tl.arange(0, K)
    alpha = tl.load(alpha_ptr + alpha_offsets)  # Registers: K
    
    # Initialize accumulator register block for output activation y
    acc = tl.zeros((BLOCK_R,), dtype=tl.float32)
    
    # Loop over columns in chunks of BLOCK_C
    for c_start in range(0, C, BLOCK_C):
        offsets_c = c_start + tl.arange(0, BLOCK_C)
        mask_c = offsets_c < C
        
        # 1. Load active activation coordinates of shape (BLOCK_C,)
        x_offset = sample_id * stride_x_b + offsets_c * stride_x_c
        x_val = tl.load(x_ptr + x_offset, mask=mask_c, other=0.0)
        
        # 2. Load static base and holographic weight blocks of shape (BLOCK_R x BLOCK_C)
        wb_offset = offsets_r[:, None] * C + offsets_c[None, :]
        w_base = tl.load(w_base_ptr + wb_offset, mask=mask_r[:, None] & mask_c[None, :], other=0.0)
        w_holo = tl.load(w_holo_ptr + wb_offset, mask=mask_r[:, None] & mask_c[None, :], other=0.0)
        
        # 3. Compute unbinding block U_b dynamically in registers using 1D carrier signatures
        # U_b = sum_k alpha_k * (r_k * c_k^T)
        u_block = tl.zeros((BLOCK_R, BLOCK_C), dtype=tl.float32)
        for k in range(0, K):
            # Load Row and Column carrier elements for task k
            rk = tl.load(keys_r_ptr + k * R + offsets_r, mask=mask_r, other=0.0)
            ck = tl.load(keys_c_ptr + k * C + offsets_c, mask=mask_c, other=0.0)
            
            # Broadcast row/col carriers to 2D block outer product and scale by alpha
            uk = rk[:, None] * ck[None, :]
            u_block += alpha[k] * uk
            
        # 4. Perform register-level weight transcription (demodulation)
        # w_dynamic = w_base + w_holo * u_block
        w_dynamic = w_base + w_holo * u_block
        
        # 5. Perform matrix-vector multiplication (multiply-accumulate)
        acc += tl.sum(w_dynamic * x_val[None, :], axis=1)
        
    # Store the output block to global memory (HBM)
    y_offset = sample_id * stride_y_b + offsets_r * stride_y_r
    tl.store(y_ptr + y_offset, acc, mask=mask_r)
\end{verbatim}

As formalized in the code, the dynamically reconstructed weight block {\tt w\_dynamic} is materialized purely within streaming multiprocessor registers and immediate vector accumulators, and is consumed on-the-fly to accumulate results. The $R \times C$ transcribed weight matrix $W_b$ is never written to global memory, reducing active dynamic device VRAM usage to exactly $O(P)$ and proving the absolute hardware viability of EHPB register-level unbinding.

\subsection{Resolving the Key Storage Memory Paradox: PRNG Seed-Based Key Generation}
\label{app:prng_key_generation}

A major practical paradox arises when transitioning from factored rank-1 carrier keys (which require only $O(K(R+C))$ key storage parameters but introduce highly structured, destructive noise collapsing classification performance to 25.4\% Joint Mean) to full-rank 2D keys (which break structured correlations and boost performance, but require $K \times R \times C$ numbers). Storing these full-rank keys in RAM requires $O(K \times P)$ memory, which is mathematically identical to storing the original task vectors $V_k$ themselves, rendering the storage-efficiency motivation of EHPB obsolete.

We propose a complete, mathematically elegant resolution to this memory paradox: \textbf{Pseudo-Random Seed-Based Key Generation}. Rather than storing full-rank 2D key matrices $K_k \in \{-1, 1\}^{R \times C}$ in HBM, we store only a single 32-bit integer random seed $S_k \in \mathbb{N}$ per task $k$, scaling the key storage memory to exactly $O(K)$ words (a few bytes).

During fused demodulation inside the Triton kernel, when a thread block is scheduled to process coordinates $(r, c)$, each thread generates the required full-rank bipolar key element $K_{k, r, c}$ on-the-fly inside local registers using a lightweight, hardware-accelerated Pseudo-Random Number Generator (PRNG) kernel (such as a Philox-4x32 or XORShift generator) seeded with $S_k$:
\begin{equation}
K_{k, r, c} = \text{PRNG}(S_k, r, c) \in \{-1, 1\}
\end{equation}
Because state-of-the-art PRNG algorithms like Philox rely on simple, fast register-level bitwise operations and integer multiplications, generating key coordinates on-the-fly adds negligible instruction-level FLOP overhead. By generating full-rank keys dynamically inside registers, we completely bypass the storage memory bottleneck. The keys are never materialized in HBM, reducing active storage memory from $O(K \times P)$ to exactly $O(K)$ integers, completely resolving the Key Storage Memory Paradox while maintaining the unstructured, high-frequency noise benefits of full-rank keys.

\subsection{Analysis of Dynamic Peak Memory and Runtime Execution Overhead}
\label{app:runtime_memory_analysis}

Another major physical bottleneck in naive dynamic ensembling is the dynamic memory footprint during parallel batch inference. Evaluating sample-wise model merging in standard PyTorch eager-mode using vectorized loops ($\mathtt{torch.vmap}$) materializes $B$ unique active weight matrices $W_b \in \mathbb{R}^{R \times C}$ in RAM. For a batch size of $B=256$, this scales dynamic memory usage to $O(B \times P)$, which is highly prone to triggering Out-of-Memory (OOM) errors on resource-constrained edge hardware.

Under our fused Triton register-level demodulation pipeline, this dynamic memory bottleneck is completely eliminated. Because demodulation and multiply-accumulate operations are fused at the instruction level inside local SM registers and SRAM, the sample-specific weights $W_b$ are never written back or materialized in global memory (HBM). The dynamic weight matrix exists only for a split second in registers, and is immediately consumed to propagate activations:
\begin{equation}
y_{b, r} = \sum_{c=1}^C \left( W_{\text{base}, r, c} + W_{\text{holo}, r, c} \cdot U_{b, r, c} \right) h_{b, c}
\end{equation}
Only the input activation $h_b \in \mathbb{R}^C$, output activation $y_b \in \mathbb{R}^R$, and the shared matrices $W_{\text{base}}, W_{\text{holo}}$ are stored in HBM. As a result, the peak global memory footprint remains strictly bounded at:
\begin{equation}
\text{Mem}_{\text{peak}} = O(P + B \times (R + C)) \text{ parameters}
\end{equation}
which scales strictly as $O(P)$ with respect to model parameters and $O(B)$ with respect to activation size. This is mathematically equivalent to the peak memory consumption of standard single-model inference, completely eliminating the $O(B \times P)$ dynamic weight memory bottleneck. 

This analysis proves that EHPB, when coupled with custom fused kernels, is physically viable for edge deployment, providing a mathematically portfolio-friendly path to on-device multi-task intelligence.

\section{Mathematical Proof of Reconstruction Accuracy}
\label{app:theorem_proof}

We provide the complete step-by-step mathematical proof of Theorem 3.1.

\begin{theorem}[Reconstruction Accuracy]
For any layer $l$ and sample $b$, the holographic demodulated parameter offset $W_{\text{holo}}^{(l)} \odot U_b^{(l)}$ satisfies:
\begin{equation}
W_{\text{holo}}^{(l)} \odot U_b^{(l)} = \sum_{k=1}^K \alpha_{k, b} V_k^{(l)} + \Xi_b^{(l)}
\end{equation}
where $\Xi_b^{(l)}$ is a high-frequency, zero-mean cross-talk noise matrix.
\end{theorem}

\begin{proof}
By substituting the definitions of $W_{\text{holo}}^{(l)} = \sum_{j=1}^K V_j^{(l)} \odot K_j^{(l)}$ and $U_b^{(l)} = \sum_{k=1}^K \alpha_{k, b} K_k^{(l)}$:
\begin{equation}
W_{\text{holo}}^{(l)} \odot U_b^{(l)} = \left( \sum_{j=1}^K V_j^{(l)} \odot K_j^{(l)} \right) \odot \left( \sum_{k=1}^K \alpha_{k, b} K_k^{(l)} \right)
\end{equation}
Expanding the double summation over $j$ and $k$:
\begin{align}
W_{\text{holo}}^{(l)} \odot U_b^{(l)} = &\sum_{k=1}^K \alpha_{k, b} V_k^{(l)} \odot \left( K_k^{(l)} \odot K_k^{(l)} \right) \nonumber \\
&+ \sum_{j \neq k} \alpha_{k, b} V_j^{(l)} \odot \left( K_j^{(l)} \odot K_k^{(l)} \right)
\end{align}

Since the carrier keys are bipolar, $K_k^{(l)} \in \{-1, 1\}^{R \times C}$. Therefore, the element-wise self-product at every coordinate $(r, c)$ is:
\begin{equation}
\left( K_k^{(l)} \odot K_k^{(l)} \right)_{r, c} = \left( K_{k, r, c}^{(l)} \right)^2 = (\pm 1)^2 = 1
\end{equation}
which implies:
\begin{equation}
K_k^{(l)} \odot K_k^{(l)} = \mathbf{1}_{R \times C}
\end{equation}
where $\mathbf{1}_{R \times C}$ is the matrix of all ones. Substituting this back into the expansion:
\begin{equation}
W_{\text{holo}}^{(l)} \odot U_b^{(l)} = \sum_{k=1}^K \alpha_{k, b} V_k^{(l)} + \sum_{j \neq k} \alpha_{k, b} V_j^{(l)} \odot \left( K_j^{(l)} \odot K_k^{(l)} \right)
\end{equation}

Let the cross-talk noise matrix $\Xi_b^{(l)}$ be defined as:
\begin{equation}
\Xi_b^{(l)} = \sum_{j \neq k} \alpha_{k, b} V_j^{(l)} \odot \left( K_j^{(l)} \odot K_k^{(l)} \right)
\end{equation}

For any $j \neq k$, the term $K_j^{(l)} \odot K_k^{(l)}$ is the element-wise product of two independent, random bipolar matrices. Since each element of $K_j^{(l)}$ and $K_k^{(l)}$ is drawn independently from the uniform distribution over $\{-1, 1\}$, the element-wise product at coordinate $(r, c)$ is:
\begin{equation}
\left( K_j^{(l)} \odot K_k^{(l)} \right)_{r, c} = K_{j, r, c}^{(l)} K_{k, r, c}^{(l)} \sim \mathcal{U}(\{-1, 1\})
\end{equation}
The expected value of each element in this product matrix is:
\begin{equation}
\mathbb{E} \left[ \left( K_j^{(l)} \odot K_k^{(l)} \right)_{r, c} \right] = \frac{1}{2}(1) + \frac{1}{2}(-1) = 0
\end{equation}
Therefore, taking the expectation of the cross-talk noise matrix $\Xi_b^{(l)}$ under the distribution of the random carrier keys:
\begin{equation}
\mathbb{E} \left[ \Xi_b^{(l)} \right] = \sum_{j \neq k} \alpha_{k, b} V_j^{(l)} \odot \mathbb{E} \left[ K_j^{(l)} \odot K_k^{(l)} \right] = \mathbf{0}_{R \times C}
\end{equation}
where $\mathbf{0}_{R \times C}$ is the matrix of all zeros. This completes the proof that EHPB reconstructs the target dynamic ensembled weight matrix up to a high-frequency, zero-mean cross-talk noise term.
\end{proof}

\section{Scaling to Large Overparameterized Foundation Models: Speculative Noise Filtering}
\label{app:foundation_scaling}

We mathematically argue that as model scale and overparameterization increase, massive foundation models are naturally far more robust to EHPB's coordinate-wise weight noise than our toy ViT-Tiny sandbox due to two physical hardware and architectural properties:

\begin{enumerate}
\item \textbf{High-Dimensional Token and Spatial Pooling:}
In modern transformer architectures, representations are processed through multi-head attention and pooling. The global visual pooling or causal attention mechanism computes a weighted spatial average of token features:
\begin{equation}
\bar{z}_b = \sum_{i=1}^N \beta_i z_{i, b}
\end{equation}
Because EHPB's weight reconstruction noise $\Xi_b^{(l)}$ is mathematically proved to be zero-mean and composed of high-frequency random sign fluctuations (Theorem 3.1), the resulting propagation error $e_b^{(l)} = \Xi_b^{(l)} h_b^{(l-1)}$ is a zero-mean coordinate-wise random vector. Under token pooling, these independent coordinate-wise fluctuations are averaged across the token dimension. By the Law of Large Numbers, this spatial pooling acts as a massive low-pass filter, averaging out the high-frequency weight-space noise and significantly recovering the original clean representation trajectory. In massive foundation models with sequence lengths $N \geq 2048$ and massive attention heads, this spatial averaging is exceptionally powerful, whereas in our tiny visual sandbox ($N=1$ globally pooled sample vector), the network has zero spatial redundancy to average out the noise, forcing the layers to absorb the full impact of coordinate-wise fluctuations.
\item \textbf{Representation Redundancy and Overparameterization:}
Large Language Models and foundation models are highly overparameterized. They contain a massive number of redundant activation paths and "sub-networks." Under weight-space noise, the representation is not disrupted along all dimensions simultaneously. In high-dimensional spaces ($D \geq 4096$), the probability of random noise aligning with the low-dimensional semantic manifold is extremely small (high-dimensional orthogonality). Overparameterized models can easily route semantic representations through redundant, unaffected neural coordinates, whereas tiny models like ViT-Tiny ($D=192$) have zero representation redundancy, making them highly fragile to parameter perturbations.
\end{enumerate}

Thus, we hypothesize that EHPB's absolute classification performance is a pessimistic lower bound. As the backbone scales up, EHPB's Joint Mean accuracy is highly likely to experience a massive boost due to scaling-induced noise resilience. However, we must explicitly acknowledge that this scaling analysis remains speculative and theoretical. Because our current empirical evaluation is confined to the simulated toy sandbox of Gaussian prototypes, we cannot guarantee that the mathematical properties of overparameterization and token pooling will translate flawlessly to actual, fine-tuned deep network weights.

\section{Generalizing to Higher-Order Tensors: Tucker and CP Carrier Key Decompositions}
\label{app:tensor_decompositions}

While our core formulation deconstructs EHPB using 2D parameter matrices $V_k \in \mathbb{R}^{R \times C}$ (such as those in linear layers), modern architectures regularly employ higher-order parameter structures, such as 4D convolutional weight tensors $W \in \mathbb{R}^{C_{\text{out}} \times C_{\text{in}} \times K_h \times K_w}$ or 3D attention weights. To scale hyperdimensional parameter superposition to higher-order tensors without encountering exponential key storage footprints, we propose generalizing EHPB using tensor rank decompositions, specifically the CANDECOMP/PARAFAC (CP) and Tucker decomposition paradigms.

\begin{enumerate}
\item \textbf{CP-Decomposition-Structured Keys (CP-EHPB):}
Under the CP decomposition paradigm, a $D$-order task tensor $V_k \in \mathbb{R}^{I_1 \times I_2 \times \dots \times I_D}$ is represented as a sum of rank-1 outer products. To construct low-memory carrier keys for such multi-dimensional spaces, we generate $D$ sets of random bipolar vectors $\{r_{k, d}^{(i)}\}$ of dimension $I_d$ for each task $k$ and rank component $i \in \{1, \dots, R\}$. The composite CP carrier key tensor $K_k \in \mathbb{R}^{I_1 \times I_2 \times \dots \times I_D}$ is structured as:
\begin{equation}
K_{k, i_1, i_2, \dots, i_D} = \text{sign} \left( \sum_{i=1}^R r_{k, 1}^{(i)}[i_1] \cdot r_{k, 2}^{(i)}[i_2] \dots r_{k, D}^{(i)}[i_D] \right)
\end{equation}
This formulation scales key storage requirements to exactly $O(K \cdot R \cdot \sum_d I_d)$ parameters, which completely bypasses the exponential $O(K \cdot \prod_d I_d)$ storage bottleneck of full-rank multidimensional keys.
\item \textbf{Tucker-Decomposition-Structured Keys (Tucker-EHPB):}
For complex weight tensors exhibiting structural correlations across spatial and channel dimensions, we can employ Tucker decomposition. Here, we define a small, shared core sign-tensor $G \in \{-1, 1\}^{r_1 \times r_2 \times \dots \times r_D}$ and $D$ task-specific projection matrices (factor matrices) $A_{k, d} \in \mathbb{R}^{I_d \times r_d}$ containing random bipolar keys. The Tucker-structured carrier key for expert $k$ is computed as:
\begin{equation}
K_k = G \times_1 A_{k, 1} \times_2 A_{k, 2} \dots \times_D A_{k, D}
\end{equation}
where $\times_d$ denotes the d-mode product. Tucker-EHPB allows practitioners to compress spatial kernels (e.g., $K_h \times K_w = 3 \times 3$) using extremely low-rank projections while dedicating higher ranks to channel dimensions, optimizing the trade-off between reconstruction noise and parameters.
\end{enumerate}

These decomposition paradigms prove that EHPB's interdisciplinary, holographic weight superposition framework is structurally compatible with arbitrary tensor orders, providing a mathematically robust path to support convolutional, attention, and mixture-of-experts parameters alike.


\section{Routing Optimization Details and Calibration Sensitivity Sweep}
\label{app:routing_optimizer_details}

In this section, we provide the precise optimizer configurations and structural details of the dynamic routing network described in Section~\ref{subsec:transcription_gating}, followed by a systematic empirical sensitivity analysis of routing performance as a function of the calibration set size.

\subsection{Routing Network Architecture and Optimizer Configurations}
The dynamic routing network $f_{\text{route}}$ (Equations 7-8) is modeled as a lightweight, single-layer linear layer mapping the unit-norm PCA projection $\psi \in \mathbb{R}^K$ to task ensembling logits. The projection matrix $W_{\text{router}}$ contains $L \times K \times K = 14 \times 4 \times 4 = 224$ parameters, and the bias vector $b_{\text{router}}$ contains $L \times K = 14 \times 4 = 56$ parameters, yielding an ultra-compact total parameter footprint of $280$ parameters.

During the post-hoc calibration phase, the router is optimized using the \textbf{AdamW} optimizer. The exact hyperparameters are detailed as follows:
\begin{itemize}
\item \textbf{Learning Rate:} $\eta = 0.01$, with flat schedule and zero warmup.
\item \textbf{Weight Decay:} $\lambda_{\text{wd}} = 10^{-3}$ applied to $W_{\text{router}}$ to prevent unconstrained logit scaling and penalize transductive overfitting.
\item \textbf{Optimization Steps:} Fixed at $100$ epochs over the entire calibration set.
\item \textbf{Batch Size:} Full-batch training (i.e., the entire $4B$ samples are processed jointly in each optimization step to compute stable multi-task cross-entropy gradients).
\end{itemize}

Crucially, the router parameters are optimized \textit{once post-hoc} on the tiny, offline calibration validation set (consisting of $B$ labeled or unlabeled samples per task) and are completely frozen thereafter. No parameters are updated at test-time (Test-Time Adaptation), which avoids the severe transductive overfitting risks and quadratic memory caching overhead associated with on-line backward passes on device. Under out-of-distribution tasks, the frozen linear router is robust because the PCA projection operator maps incoming activations onto the pre-established $K$-dimensional task coordinate space, acting as a structural low-pass filter that prevents the router from being swayed by high-frequency input drift.

\section{EHPB and Quantization: High-Frequency Noise under 4-bit and 8-bit Boundaries}
\label{app:quantization_boundaries}

On-device edge deployment almost universally relies on model quantization, transforming 32-bit floating-point (FP32) weights into low-bit formats, such as 8-bit integer (INT8) or 4-bit integer (INT4). In this section, we analyze the theoretical and physical interaction between EHPB's high-frequency holographic reconstruction noise and low-bit quantization boundaries, exposing key systems-level challenges and proposing robust mitigations.

\subsection{Quantization Clipping and Precision Underflow Paradox}
Standard uniform quantization maps a continuous range of weights $[W_{\text{min}}, W_{\text{max}}]$ to a discrete set of integers. For instance, asymmetric $b$-bit quantization represents a weight as:
\begin{equation}
W_{\text{quant}} = \text{clamp} \left( \text{round} \left( \frac{W}{\Delta} \right) + Z, 0, 2^b - 1 \right)
\end{equation}
where $\Delta = (W_{\text{max}} - W_{\text{min}}) / (2^b - 1)$ is the scale factor and $Z$ is the zero-point. Under EHPB, the superimposed holographic matrix is $W_{\text{holo}} = \sum_j V_j \odot K_j$. The unbinding process demodulates weights on-the-fly as $W_b = W_{\text{base}} + W_{\text{holo}} \odot U_b$. 

If we quantize the stored holographic matrix $W_{\text{holo}}$ to INT8 or INT4, we encounter a severe \textit{Precision Underflow Paradox} due to hyperdimensional superposition:
\begin{itemize}
\item \textbf{Noise-Floor Amplification:} The holographic matrix $W_{\text{holo}}$ is a sum of $K$ independent task vectors. Because the random keys $K_j$ have values in $\{-1, 1\}$, $W_{\text{holo}}$ is mathematically equivalent to the target weight vector $V_k$ plus a zero-mean high-frequency cross-talk noise matrix $\Xi \in \mathbb{R}^{R \times C}$ with variance proportional to $K-1$. The maximum and minimum boundaries of $W_{\text{holo}}$ are heavily expanded by this cross-talk noise: $W_{\text{holo, max}} \approx \sqrt{K} V_{\text{max}}$.
\item \textbf{Quantization Step Expansion:} Because the range $[W_{\text{min}}, W_{\text{max}}]$ expands symmetrically by a factor of $\sqrt{K}$ due to superposition noise, the quantization step size $\Delta_{\text{holo}}$ becomes much larger. 
\item \textbf{Catastrophic Underflow:} Small-magnitude, high-signal components of the task vectors $V_k$ are smaller than the expanded step size $\Delta_{\text{holo}}$. Consequently, when $W_{\text{holo}}$ is quantized to INT8 or INT4, these critical weights are rounded to zero (precision underflow). When unbinding is performed, instead of retrieving the true signal, the network demodulates a highly degraded, zeroed-out sparse weight tensor, collapsing the classification performance completely.
\end{itemize}

\subsection{Proposing Quantization-Aware Holographic Superposition (QA-EHPB)}
To resolve the underflow paradox and enable stable edge quantization, we propose three interdisciplinary mitigations:
\begin{enumerate}
\item \textbf{Quantization-Aware Binding (QAB):} Rather than quantizing post-superposition, we perform quantization-aware training (QAT) of the individual expert task vectors $V_k$. During fine-tuning, we apply fake-quantization operators directly to the holographic weight:
\begin{equation}
W_{\text{holo}}^{\text{quant}} = \text{Quantize} \left( \sum_j V_j \odot K_j \right)
\end{equation}
This forces the individual expert weights to adapt and find optimal parameters that are robust to both hyperdimensional superposition noise and the quantization scale.
\item \textbf{Demodulation-First Kernel Fusion (INT8-FP16 hybrid):} To avoid quantizing the noisy $W_{\text{holo}}$, we store $W_{\text{holo}}$ and the carrier keys in low-bit INT8 formats, but perform the unbinding multiplication $W_{\text{holo}} \odot U_b$ inside the GPU shared memory (SRAM) registers in higher precision FP16. This preserves the on-device memory savings of storing INT8 weights, while completely avoiding quantization errors during demodulation.
\item \textbf{Task-Wise Dynamic Scaling:} Because different tasks $j$ exhibit varying gradient magnitudes, we introduce a learnable, task-specific scalar scale factor $s_j > 0$. The holographic matrix is constructed as $W_{\text{holo}} = \sum_j s_j V_j \odot K_j$. During calibration, the scale factors are optimized to pack and balance task weight ranges within the discrete INT8/INT4 bins, maximizing dynamic range utilization.
\end{enumerate}

These quantization insights provide edge-systems engineers with a concrete, mathematically rigorous blueprint to deploy hyperdimensional parameter-space merging on actual physical embedded chips with low-bit memory layouts, neutralizing the trade-off between quantization error and superposition noise.
