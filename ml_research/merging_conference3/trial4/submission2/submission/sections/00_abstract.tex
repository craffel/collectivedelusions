\begin{abstract}
On-device multi-task ensembling via weight-space model merging has emerged as a compelling, zero-overhead paradigm for deploying specialized models on resource-constrained edge hardware. However, deploying merged models in the wild introduces a critical, unaddressed bottleneck: edge accelerators and compilers (e.g., mobile TPUs, DSPs, automotive ASICs) employ highly heterogeneous, incompatible post-training quantization (PTQ) standards. Existing quantization-aware model merging methods, such as Q-Merge, optimize blending coefficients under a single simulated operator, but we show they suffer from cross-schema sensitivity, where learned coefficients underperform when deployed on mismatched target hardware accelerators. 

To resolve this fundamental bottleneck, we introduce \textbf{OmniMerge}, a training-free, multi-schema stochastic co-optimization framework for robust model merging. OmniMerge addresses hardware heterogeneity via two primary mechanisms: (1) \textbf{Stochastic Operator Sampling (SOS)}, which stochastically samples active quantization operators from a discrete pool during test-time adaptation, acting as parameter-space data augmentation to prevent boundary-overfitting; and (2) \textbf{Scale and Zero-Point Noise Perturbation (SZNP)}, which injects noise into the dynamic rounding grid to smooth the non-differentiable loss landscape. Evaluated on a Vision Transformer backbone (\texttt{ViT-Tiny}) across MNIST, FashionMNIST, CIFAR-10, and SVHN, OmniMerge completely closes the cross-schema generalization gap. Under robust 8-bit post-training quantization, OmniMerge achieves up to \textbf{50.78\%} average accuracy, outperforming both Q-Merge and Quantized AdaMerging across all 5 mismatched target compilers, establishing a new state-of-the-art for hardware-invariant, zero-overhead edge ensembling.
\end{abstract}
