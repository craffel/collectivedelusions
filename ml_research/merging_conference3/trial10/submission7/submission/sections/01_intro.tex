\section{Introduction}
\label{sec:intro}

Parameter-Efficient Fine-Tuning (PEFT) techniques, such as Low-Rank Adaptation (LoRA) \cite{hu2021lora} and task vectors \cite{ilharco2022editing}, have transformed the deployment of deep learning models. Rather than deploying separate, costly, large-scale models for every task, systems administrators can serve a single frozen base model alongside a fleet of compact task-specific expert adapters. To handle incoming queries that dynamically shift over time, test-time dynamic model merging and expert ensembling have emerged as critical techniques \cite{liang23sable, choshen22fusing}. Under these frameworks, intermediate representations are extracted at early layers, projected onto task-specific subspaces to extract task coordinates, and dynamically blended using a routing policy to select and ensemble the best adapters in a single forward pass.

To suppress sample-level feature noise and prevent high-frequency routing fluctuations, stateful or kinetic routers have recently been introduced, such as ChemMerge \cite{chemmerge25} and PAC-Kinetics \cite{pac_zca_2026}. By modeling the ensembling routing trajectory as a continuous-time chemical kinetics reaction or a stable first-order diagonal recurrence, these stateful approaches act as temporal low-pass filters. They smooth the ensembling weights $\boldsymbol{\alpha}_t$ across consecutive queries, resulting in a dramatic reduction in routing jitter and ensuring representational stability across deep layers.

Despite their mathematical elegance, existing stateful routing frameworks suffer from a fatal real-world deployment blocker: **they assume incoming queries belong to a continuous, highly correlated single-user stream.** In actual production environments, such as high-throughput LLM serving systems, edge-compute gateways, or multi-user web servers, workloads are rarely homogeneous or isolated. Instead, servers process **multi-tenant interleaved streams**, where queries from independent users/sessions with vastly different underlying tasks arrive in rapid, interleaved successions (e.g., User A submits a medical query, User B submits a programming query, User A submits another medical query).

When standard stateful routers are deployed in these multi-tenant environments, they suffer from what we define as the **state contamination (cross-talk) bottleneck**. Because they maintain a single, global routing state vector $\mathbf{s}_t$, the routing memory is blended across unrelated tenants. The temporal smoothing state left by User B's CIFAR query directly corrupts the routing decision for User A's subsequent MNIST query. This cross-tenant state contamination leads to severe routing lag, activates incorrect expert adapters, induces representational bleeding, and causes catastrophic drops in serving accuracy (up to 9.0\% absolute classification accuracy). Consequently, under realistic deployment workloads, standard stateful ensembling performs worse than simple stateless baselines.

In this work, we adopt a strictly **pragmatic perspective** to resolve this fundamental deployment blocker. We argue that for stateful model merging to be viable in production systems, it must be robust to multi-tenant workloads while maintaining sub-microsecond latency and negligible memory footprints. To achieve this, we introduce **Tenant-Decoupled Stateful Routing (TDSR)**, also known as **Slot-Kinetics**.

Instead of a single global state, TDSR maintains a highly compact pool of virtual routing state slots. When a query is received, it is mapped to its corresponding virtual slot, isolating the temporal smoothing dynamics within respective tenant contexts. We present two highly practical modes of session routing:
\begin{enumerate}
    \item \textbf{Explicit Session Tagging (Metadata-Tagged):} If the serving framework (e.g., S-LoRA \cite{s-lora} or Punica \cite{chen2023punica}) provides session metadata or tenant IDs, the router retrieves and updates the corresponding state slot directly with zero computational overhead.
    \item \textbf{Implicit Tagless Clustering (Dynamic Inference):} When metadata is unavailable, the router dynamically infers the session context on-the-fly. By computing the online cosine similarity between the current query coordinate vector and fixed orthogonal coordinate detector centroids ($\mathbf{c}_{m, m} = 1.0$), the router assigns the query to the most specialized slot. This completely eliminates centroid drift and runaway slot attraction (clustering collapse), ensuring perfect unsupervised slot specialization without offline calibration.
\end{enumerate}

To release slot memory and prevent stale states from accumulating indefinitely, all inactive slots are governed by logical session-step decay or dual-clock physical decay. Once the active slot is selected, its state is updated via a stable recurrence and mapped to ensembling weights via a multi-temperature Gibbs Softmax policy.

We evaluate TDSR inside the high-fidelity Analytical Coordinate Sandbox (ICS). Under a rapidly interleaved (i.i.d.) multi-tenant streaming workload, our experiments demonstrate that:
\begin{itemize}
    \item Standard stateful routing (Global PAC-Kinetics) under interleaved serving workloads loses up to 5.00\% absolute accuracy compared to the Oracle clean stream (which operates on clean, non-interleaved task sequences). This highlights the severe impact of cross-tenant state contamination.
    \item Our proposed TDSR Explicit successfully resolves state contamination, achieving up to \textbf{70.60\%} classification accuracy on Orthogonal Manifolds and \textbf{70.85\%} on Overlapping Manifolds. This represents a significant \textbf{+1.90\%} absolute accuracy improvement over the contaminated Global baseline on Orthogonal Manifolds, and performing within 0.50\% of the isolated clean-stream baseline ceiling on Overlapping Manifolds. Crucially, TDSR Explicit stabilizes ensembling trajectories, slashing intra-session routing jitter by up to \textbf{2.4$\times$} relative to stateless SABLE.
    \item In terms of routing jitter, we expose a critical scientific phenomenon: under rapidly interleaved (i.i.d.) streams, global inter-session jitter remains high ($\approx 0.91 - 1.03$) to track rapid task context-switches, whereas our proposed intra-session jitter analysis demonstrates that TDSR successfully maintains temporal smoothing within each session context.
    \item In terms of system overhead, maintaining a small state pool requires a microscopic $M \times K$ tensor array. Execution involves basic vector operations that complete in less than 1.5 microseconds, introducing zero database lookups, disk lookups, or network overhead.
\end{itemize}

Our contributions establish a highly practical, robust, and mathematically sound framework for deploying stable stateful dynamic model merging in high-throughput multi-tenant production infrastructures.
