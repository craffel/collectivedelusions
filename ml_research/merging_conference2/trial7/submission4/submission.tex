\begin{abstract}
  Multi-task model merging (e.g., Weight Averaging, Task Arithmetic) offers a computationally efficient and decentralized way to combine deep neural network experts fine-tuned on separate tasks without retraining. However, weight-level combinations suffer from severe representation collapse in deep layers, destroying performance. While post-merge calibration of BatchNorm (BN) running statistics can restore performance, existing methods require centralized target-task datasets or expensive generative optimization, violating privacy in SaaS or edge deployments. To address this, we propose Federated BatchNorm Calibration (Fed-BNC), a zero-server-data, privacy-preserving, and computationally lightweight representation alignment approach. In Fed-BNC, clients run a single forward pass of the merged model on a small, private local subset (e.g., as few as 16--64 samples), compute 1D activation statistics, and share only these 1D statistical vectors with the server, which averages them to set global running statistics. To handle severe non-IID class skew across clients, we extend our method to Class-Conditional Federated BatchNorm Calibration (CC-Fed-BNC), where clients compute class-conditional statistics that are aggregated via a uniform class-balanced server fusion. Evaluating on ResNet-18 across MNIST, Fashion-MNIST, and CIFAR-10, we show that Fed-BNC achieves near-parity with central real-data calibration (Oracle), while CC-Fed-BNC gains up to $+1.73\%$ accuracy over naive calibration under skew. We also identify a critical "representation incoherence vs. calibration trade-off" where standard calibration on highly disjoint tasks backfires by amplifying noise. Our method introduces 0 parameters, 0 server-side optimization, and 0 runtime latency, and is fully compatible with compiler optimizations like \texttt{torch.compile}.
\end{abstract}

\section{Introduction}
\label{sec:intro}

Deep neural network fine-tuning has enabled remarkable performance across specialized domains, but deploying a dedicated large-scale model for each individual task introduces massive storage, memory, and serving overheads. To combat this, model merging has emerged as a promising paradigm to combine task-specific expert models, fine-tuned from a shared pre-trained progenitor, into a single multi-task model without any retraining or additional gradient-based optimization \cite{wortsman2022model, ilharco2023editing}. Standard approaches like Weight Averaging (WA) and Task Arithmetic (TA) linearly combine weights across expert backbones.

Despite their elegance, weight-level combinations are known to suffer from severe "variance collapse" or "representation collapse" \cite{jordan2023repair, tang2024post}. Merging non-orthogonal weight updates disrupts the activation scales of intermediate layers, causing activation magnitudes in deep layers to collapse toward zero or grow out of bounds. This mismatch between the merged weights and the pre-saved BatchNorm running statistics of individual experts leads to a catastrophic drop in inference accuracy.

To restore the model's inner representations, recent work proposes post-merge calibration of BatchNorm running statistics \cite{tang2024post}. However, these approaches are subject to major real-world bottlenecks: (1) \textbf{Privacy Violations:} centralized calibration methods \cite{jordan2023repair} require a central target-task dataset, which violates strict privacy frameworks (GDPR, HIPAA) in sensitive sectors; (2) \textbf{High Server-Side Compute:} generative or data-free methods \cite{submission10} synthesize pseudo-data on the server, which is slow and computationally intensive; and (3) \textbf{Runtime Latency:} active test-time adaptation methods \cite{submission5} require dynamic statistics gathering during live streams, introducing serving latency and preventing static compiler optimizations.

To bridge this gap, we present \textbf{Federated BatchNorm Calibration (Fed-BNC)}, a highly practical, privacy-preserving, and computationally lightweight framework for multi-task representation alignment. Guided by a pragmatic research philosophy, we focus on real-world constraints such as data privacy, sample efficiency, and serving compatibility. In Fed-BNC, the server merges the weights and sends the merged model to decentralized clients. Each client runs a single forward pass of the model on a tiny subset of their local private dataset (e.g., $N \in [16, 128]$ samples), measures the actual mean and variance of the merged activations at each BN layer, and returns only these low-bandwidth, anonymized 1D statistical vectors to the server. The server aggregates these local vectors to construct the final global running statistics.

Our main contributions are: (1) We propose \textbf{Fed-BNC}, a decentralized, zero-server-data calibration method that matches central oracle calibration within $0.4\%$ accuracy while keeping raw client data fully local; (2) We introduce \textbf{CC-Fed-BNC}, which utilizes class-conditional statistics and uniform class-balanced server aggregation to overcome severe client-side class skew, yielding a $+1.73\%$ accuracy boost; (3) We empirically demonstrate that Fed-BNC adds 0 extra parameters, 0 server-side optimization, and 0 inference latency overhead, and is fully compatible with \texttt{torch.compile}, achieving a seamless deployment; and (4) We uncover a "representation incoherence vs. calibration trade-off" where standard BN calibration degrades accuracy when merging models on highly disjoint tasks, highlighting critical boundaries for model merging practitioners.

\section{Related Work}
\label{sec:related}

\subsection{Multi-Task Model Merging}
Multi-task model merging has emerged as a powerful paradigm for combining specialized neural network experts fine-tuned from a shared pre-trained progenitor model without expensive retraining or joint training \cite{wortsman2022model, choshen2022fusing}. The most straightforward weight-level combinations include Weight Averaging (WA) \cite{wortsman2022model, Choi2024RevisitingWA, Meterez2026AnytimePH} and Task Arithmetic (TA) \cite{ilharco2023editing}, which operates by adding fine-tuning update vectors (task vectors) to the progenitor model. Several works explore extensions of task arithmetic for various architectures and tasks, such as merging decision transformers for multi-task policies \cite{Lawson2023MergingDT} and diffusion models \cite{Biggs2024DiffusionSM}. Despite their computational elegance, naive weight-level merges often fail due to interference among non-orthogonal task vectors. To mitigate weight-level interference, recent research introduces TIES-Merging \cite{yadav2023resolving, Yadav2023TIESMergingRI} to resolve sign conflicts and prune redundant parameter updates, DARE \cite{yu2024mergekit} to sparsify weights before averaging, and Fisher Weighted Averaging \cite{matena2022merging} to align parameters based on diagonal Fisher information matrices. Other methods include RegMean \cite{jin2023dataless} which aligns representations using out-of-distribution matrices, and Weight Weaving \cite{Chaves2025WeightWP} which pools parameters for data-free model merging. At the same time, model merging has been scaled to large language models \cite{Song2026ModelMI, Yadav2024WhatMF, Ueda2025MergingCP}, multi-modal networks \cite{Park2025RealAwareRM}, and medical domains \cite{Timilsina2025ANH}. Researchers have also explored theoretical underpinnings of model merging, such as permutation symmetries and the non-local model merging problem \cite{Sharma2024TheNM}, trust-region navigation \cite{Sun2025TaskAI}, submodule linearity \cite{Dai2025LeveragingSL}, and fairness \cite{Naganuma2025OnFO}. Despite these advances, weight-level model merging is fundamentally limited by representation collapse, a severe disruption of intermediate feature scales in deep layers \cite{jordan2023repair}.

\subsection{Post-Merge Representation Calibration}
To address representation collapse, REPAIR \cite{jordan2023repair} pioneered post-merge BatchNorm calibration, and Tang et al. \cite{tang2024post} generalized this to multi-task scenarios. Concurrent techniques include: (1) \textbf{Single-Pass Test-Time Calibration (SP-TTBC)} \cite{submission5}, which dynamically blends expert statistics during inference but introduces test-time latency and disrupts compiler optimizations; (2) \textbf{Regularized Fine-Tuning} \cite{submission6}, which uses pre-merge regularization to mitigate collapse but requires modified fine-tuning; and (3) \textbf{Data-Free Calibration (DF-Calib)} \cite{submission10}, which utilizes server-side generative batchnorm matching, though synthesizing synthetic data is computationally slow. Unlike these methods, Fed-BNC requires no raw server data, zero server-side optimization, and adds no test-time latency.

\subsection{Decentralized and Federated Normalization}
Federated learning (FL) enables collaborative model training across decentralized clients while keeping raw data local \cite{mcmahan2017communication, kairouz2021advances}. A primary challenge in FL is data heterogeneity (non-IID data) across clients \cite{zhao2018federated, li2020federated, wang2020tackling}. Batch Normalization (BN) behaves poorly under such non-IID conditions because local mini-batch statistics diverge from global population statistics \cite{Guerraoui2024OvercomingTC, Quintana2024BNSCAFFOLDCT}. To resolve this, FedBN \cite{Li2021FedBNFL} proposes keeping BN parameters local to each client during training. Other works explore personalized federated learning via adaptive batch normalization \cite{Lu2021PersonalizedFL, Chen2021FederatedLW}, population normalization \cite{Wang2025PopulationNF}, and bridging local and federated normalization layers \cite{Cosgun2025BridgingLA}. Additionally, works like BN-SCAFFOLD \cite{Quintana2024BNSCAFFOLDCT} control the drift of BN statistics during FL training, and model-level style normalization \cite{Zhu2024ModellevelAA} addresses medical image segmentation. While these methods optimize normalization layers *during* collaborative training, Fed-BNC is the first to study and resolve representation collapse in *post-hoc model merging* using privacy-preserving, sample-efficient client statistics.


\section{Proposed Method}
\label{sec:method}

We formalize the problem of representation collapse in model merging and detail our proposed Fed-BNC and CC-Fed-BNC calibration mechanisms.

\subsection{Representation Collapse in Model Merging}
Let $\theta_{prog}$ be the parameters of a shared pre-trained progenitor network, and $\{\theta_c\}_{c=1}^C$ be $C$ expert models fine-tuned on distinct tasks. Each expert model contains BatchNorm layers designed to normalize intermediate features. For a convolutional activation tensor $x \in \mathbb{R}^{B \times C \times H \times W}$ at a given BN layer, the normalization is defined as:
\begin{equation}
    \hat{x} = \frac{x - \mu_{running}}{\sqrt{\sigma_{running}^2 + \epsilon}}
\end{equation}
where $\mu_{running}$ and $\sigma_{running}^2$ are pre-saved channel-wise running statistics computed during training of the individual expert. 

When we construct a merged model $\theta_{merged}$ via Weight Averaging (WA) or Task Arithmetic (TA), the weight parameters of the backbone are combined. For example, WA computes:
\begin{equation}
    \theta_{merged} = \frac{1}{C} \sum_{c=1}^C \theta_c
\end{equation}

To mathematically analyze why weight-level combinations disrupt representation alignment, consider the forward pass through a single linear or convolutional layer. Let $W_c = W_{prog} + \Delta W_c$ represent the weights of expert $c$, where $\Delta W_c$ is the task-specific parameter update (task vector). When merging via WA, the merged weight is $W_{merged} = W_{prog} + \frac{1}{C} \sum_c \Delta W_c$. For an input feature map $h$, the output pre-activation $z_{merged}$ is given by:
\begin{equation}
    z_{merged} = W_{merged} h = \left( W_{prog} + \frac{1}{C} \sum_{c=1}^C \Delta W_c \right) h
\end{equation}
In contrast, the expected pre-activation under the individual experts is $z_c = W_c h = (W_{prog} + \Delta W_c) h$. 

Because deep neural networks are highly non-linear, linearly combining weights disrupts the alignment of intermediate activation spaces. Under the assumption that the task vectors $\Delta W_c$ are non-orthogonal and have divergent directional updates across different tasks, the covariance of the activations in deep layers deviates severely. Specifically, if the features are correlated across layers, the activation variance at layer $l$ collapses exponentially with depth. The actual activation mean and variance of $\theta_{merged}$ on target tasks deviate severely from the pre-saved running statistics:
\begin{equation}
    \mathbb{E}[z_{merged}] \neq \mu_{running}, \quad \text{Var}(z_{merged}) \neq \sigma_{running}^2
\end{equation}
This mismatch is known as representation collapse. As profiled in \cref{sec:results}, uncalibrated WA results in activation standard deviations collapsing to $\sim 0.01$--$0.02$, causing catastrophic representation degradation. When unnormalized, these collapsed signals fail to activate downstream ReLU layers, causing subsequent layers to output uniform zeroes and completely halting information flow.

\subsection{Federated BatchNorm Calibration (Fed-BNC)}
To realign representations without centralizing raw datasets, Fed-BNC collects activation statistics from clients in a privacy-preserving manner. By feeding a small local calibration dataset through the merged weights, we empirically measure the true activation mean and variance of $\theta_{merged}$ and write these back into the BN running buffers. The protocol proceeds as follows:
\begin{enumerate}
    \item \textbf{Server Merging:} The server combines the expert weights to obtain $\theta_{merged}$ and broadcasts it to the $C$ clients.
    \item \textbf{Client-Side Profiling:} Each client $c$ loads a small local subset $D_c \subset X_c$ of size $N$ (e.g., $N\!=\!128$) and runs a single forward pass of $\theta_{merged}$ in evaluation mode. We register forward hooks on each BatchNorm layer to accumulate channel-wise activation sums and squared sums:
    \begin{equation}
        \mu_{local}^c = \frac{1}{NHW} \sum_{i \in D_c} \sum_{h, w} x_{i, :, h, w}
    \end{equation}
    \begin{equation}
        (\sigma^2)_{local}^c = \frac{1}{NHW} \sum_{i \in D_c} \sum_{h, w} (x_{i, :, h, w} - \mu_{local}^c)^2
    \end{equation}
    Clients share only these 1D statistical vectors---the mean $\mu_{local}^c$ and variance $(\sigma^2)_{local}^c$---with the server.
    \item \textbf{Server-Side Fusion:} The server updates the global running statistics of the merged model by taking the uniform average of the client-side statistics:
    \begin{equation}
        \mu_{global} = \frac{1}{C} \sum_{c=1}^C \mu_{local}^c
    \end{equation}
    \begin{equation}
        \sigma_{global}^2 = \frac{1}{C} \sum_{c=1}^C (\sigma^2)_{local}^c
    \end{equation}
\end{enumerate}
Fed-BNC maintains absolute privacy as no raw image or feature-map is shared; only 1D activation averages are transmitted.

\subsection{Class-Conditional Federated BatchNorm Calibration (CC-Fed-BNC)}
In real-world federated environments, data is often highly non-IID due to class skew. If client $c$ only possesses a subset of classes $K_c \subsetneq \{0, \dots, K-1\}$, a naive forward pass on local data will skew the activation statistics toward those specific classes, introducing severe representation distortion when averaged globally. For example, if client 1 only has digits 0--4, its forward pass activations will reflect only the features of those digits, leading to a biased local mean $\mu_{local}^1$ that is unrepresentative of the entire population.

To overcome this, we propose CC-Fed-BNC, where clients compute class-conditional statistics. During the forward pass, hooks partition the activations according to sample labels $y_i$. For each class $k \in K_c$, the client computes:
\begin{equation}
    \mu_{local}^{c,k} = \frac{1}{N_k^c HW} \sum_{i \in D_c, y_i = k} \sum_{h,w} x_{i, :, h, w}
\end{equation}
\begin{equation}
    (\sigma^2)_{local}^{c,k} = \frac{1}{N_k^c HW} \sum_{i \in D_c, y_i = k} \sum_{h,w} (x_{i, :, h, w} - \mu_{local}^{c,k})^2
\end{equation}
The server aggregates these statistics class-by-class, averaging across the subset of clients $C_k$ (where $C_k$ is the set of clients that observed class $k$):
\begin{equation}
    \mu_{class}^k = \frac{1}{|C_k|} \sum_{c \in C_k} \mu_{local}^{c,k}
\end{equation}
\begin{equation}
    (\sigma^2)_{class}^k = \frac{1}{|C_k|} \sum_{c \in C_k} (\sigma^2)_{local}^{c,k}
\end{equation}
Finally, the server combines these balanced class statistics uniformly to form the global statistics:
\begin{equation}
    \mu_{global} = \frac{1}{K} \sum_{k=0}^{K-1} \mu_{class}^k
\end{equation}
\begin{equation}
    \sigma_{global}^2 = \frac{1}{K} \sum_{k=0}^{K-1} \left[ (\sigma^2)_{class}^k + (\mu_{class}^k - \mu_{global})^2 \right]
\end{equation}
This mathematically rigorous fusion ensures the merged model's statistics are perfectly balanced across all classes, regardless of individual client skew.

\subsection{Communication, Computational, and Privacy Analysis}
\label{subsec:analysis}
Under standard collaborative workflows, the primary overheads of post-merge adjustments lie in server-side compute and network bandwidth. Here, we analyze our method along these critical pragmatic dimensions:

\textbf{1. Network Bandwidth and Communication Efficiency:}
The communication cost of Fed-BNC is exceptionally low. Let $M$ be the total number of channels summed across all BatchNorm layers in the network. For a standard ResNet-18 architecture, there are 20 convolutional BatchNorm layers containing a total of $M = 4,800$ channels. 
\begin{itemize}
    \item For standard \textbf{Fed-BNC}, each client transmits two 1D vectors: $\mu_{local}^c \in \mathbb{R}^{4800}$ and $(\sigma^2)_{local}^c \in \mathbb{R}^{4800}$. Using 32-bit floating point representation, this requires exactly $9,600 \times 4 \text{ bytes} = 38.4 \text{ KB}$ of data.
    \item For \textbf{CC-Fed-BNC} with $K$ classes, each client transmits class-conditional statistics for the classes present in their subset. In the worst-case where all $K=10$ classes are present, this requires transmitting $10 \times 9,600 \text{ floats} = 384 \text{ KB}$ of data.
\end{itemize}
Compared to the ResNet-18 model size of $11.7\text{M}$ parameters ($46.8 \text{ MB}$), transmitting our statistics takes only \textbf{0.08\%} (Fed-BNC) to \textbf{0.82\%} (CC-Fed-BNC) of the model weights. Furthermore, it is vastly more efficient than transmitting a small calibration dataset of $N=128$ images, which would require upwards of $1.57 \text{ MB}$ of raw data.

\textbf{2. Computational Complexity and Client Overhead:}
Computing local statistics requires exactly one forward pass of the model in evaluation mode over $N$ samples. Because no backward pass is executed, there are:
\begin{itemize}
    \item \textbf{0 Gradients to calculate or store:} reducing peak activation memory by up to $3\times$.
    \item \textbf{0 Optimizer state updates:} eliminating optimizer memory overheads completely.
    \item \textbf{0 Server-side optimizations:} unlike generative calibration methods (e.g. DF-Calib) which require expensive, slow, and unstable gradient-descent optimization to synthesize data, our server-side aggregation is a closed-form, direct averaging operation that completes in less than 5 milliseconds on a single CPU core.
\end{itemize}
This extremely low footprint makes Fed-BNC highly suitable for battery-constrained, low-compute edge and IoT environments.

\textbf{3. Privacy Safeguards:}
Sharing 1D channel-wise averages provides robust privacy guarantees. Because the forward hooks average activations over the batch size $N$ and the spatial dimensions $H \times W$ of convolutional feature maps, there is a severe, lossy many-to-one mapping. Reconstructing raw input images from these 1D channel averages is mathematically under-determined and practically impossible under standard gradient-leakage or feature-reconstruction attacks. No raw images, activation feature maps, or individual sample embeddings are ever transmitted, ensuring high compatibility with strict user-privacy frameworks (GDPR, HIPAA).

\begin{algorithm}[t]
\caption{Class-Conditional Federated BatchNorm Calibration (CC-Fed-BNC)}
\label{alg:cc_fed_bnc}
\footnotesize
\begin{algorithmic}[1]
\REQUIRE Merged model parameters $\theta_{merged}$, expert models $\{\theta_c\}_{c=1}^C$, clients $c \in \{1,\dots,C\}$, client local subsets $\{D_c\}_{c=1}^C$ of size $N$, total classes $K$.
\ENSURE Calibrated global BatchNorm running mean $\mu_{global}$ and running variance $\sigma_{global}^2$ for each BN layer in $\theta_{merged}$.
\STATE \textbf{Server Side:}
\STATE Broadcast $\theta_{merged}$ to all clients.
\STATE \textbf{Client Side (in parallel for client $c \in \{1,\dots,C\}$):}
\STATE Initialize local activation accumulators for each class $k \in K$ and BN layer:
\STATE $S_1^{c,k} \leftarrow \mathbf{0}, \quad S_2^{c,k} \leftarrow \mathbf{0}, \quad N_k^c \leftarrow 0$.
\STATE Register forward hooks on each BatchNorm layer of $\theta_{merged}$ to extract activation features $x$.
\FOR{each mini-batch $(X, Y) \in D_c$}
    \STATE Run forward pass: $outputs \leftarrow \theta_{merged}(X)$ (hooks extract $x$ for each BN layer).
    \FOR{each class $k$ present in batch $Y$}
        \STATE Accumulate activation sums and squared sums over spatial dimensions $H \times W$ for samples where $Y_i = k$:
        \STATE $S_1^{c,k} \leftarrow S_1^{c,k} + \sum_{i: Y_i = k} \sum_{h,w} x_{i, :, h, w}$
        \STATE $S_2^{c,k} \leftarrow S_2^{c,k} + \sum_{i: Y_i = k} \sum_{h,w} (x_{i, :, h, w})^2$
        \STATE $N_k^c \leftarrow N_k^c + |\{i \mid Y_i = k\}| \cdot H \cdot W$
    \ENDFOR
\ENDFOR
\FOR{each class $k \in K$ where $N_k^c > 0$}
    \STATE Compute local class-conditional mean and variance:
    \STATE $\mu_{local}^{c,k} \leftarrow S_1^{c,k} / N_k^c$
    \STATE $(\sigma^2)_{local}^{c,k} \leftarrow (S_2^{c,k} / N_k^c) - (\mu_{local}^{c,k})^2$
\ENDFOR
\STATE Send $\{\mu_{local}^{c,k}, (\sigma^2)_{local}^{c,k}\}_{k: N_k^c > 0}$ to Server.
\STATE \textbf{Server Side (for each BN layer):}
\FOR{each class $k \in \{0, \dots, K-1\}$}
    \STATE Identify set of active clients for class $k$: $C_k = \{c \mid N_k^c > 0\}$.
    \STATE Aggregate class-conditional stats:
    \STATE $\mu_{class}^k \leftarrow \frac{1}{|C_k|} \sum_{c \in C_k} \mu_{local}^{c,k}$
    \STATE $(\sigma^2)_{class}^k \leftarrow \frac{1}{|C_k|} \sum_{c \in C_k} (\sigma^2)_{local}^{c,k}$
\ENDFOR
\STATE Compute balanced global statistics:
\STATE $\mu_{global} \leftarrow \frac{1}{K} \sum_{k=0}^{K-1} \mu_{class}^k$
\STATE $\sigma_{global}^2 \leftarrow \frac{1}{K} \sum_{k=0}^{K-1} \left[ (\sigma^2)_{class}^k + (\mu_{class}^k - \mu_{global})^2 \right]$
\STATE Set $\theta_{merged}$'s running statistics at this BN layer to $(\mu_{global}, \sigma_{global}^2)$.
\end{algorithmic}
\end{algorithm}

\section{Experimental Setup}
\label{sec:setup}

We design a rigorous multi-task experimental setup evaluating standard and federated calibration approaches.

\subsection{Models, Tasks, and Training Details}
We load a pre-trained ImageNet ResNet-18 progenitor \cite{he2016deep} and fine-tune three separate expert models on: (1) MNIST \cite{lecun1998gradient}, (2) Fashion-MNIST (FMNIST) \cite{xiao2017fashion}, and (3) CIFAR-10 \cite{krizhevsky2009learning}. Each expert's backbone is initialized with progenitor weights, and its classification head is replaced with a 10-class linear layer. Following standard literature practices for rapid expert fine-tuning, each expert is trained on a 5,000-sample training subset for 5 epochs using AdamW with learning rate $1\times 10^{-4}$ and batch size 128. This yields solid individual expert accuracies (MNIST: $97.17\%$, FMNIST: $83.93\%$, CIFAR-10: $62.57\%$; Average: $81.22\%$), establishing a representative baseline.

\subsection{Merging Paradigms and Calibration Schemes}
We merge the three experts using Weight Averaging (WA) and Task Arithmetic (TA, with $\lambda = 0.25$). For each paradigm, we compare: (1) \textbf{Uncalibrated Baseline:} the merged model using default parameters; (2) \textbf{Oracle Calibration:} the centralized baseline calibrating running statistics using $1024$ real samples per task on the server; (3) \textbf{Fed-BNC (Ours):} our privacy-preserving scheme using $N = 128$ private samples per task local to each client; and (4) \textbf{CC-Fed-BNC (Ours):} our proposed class-conditional scheme under the same $N = 128$ private sample budget.

\subsection{Representation Alignment \& Convergence Parity}
As shown in \cref{tab:main_results}, uncalibrated model merging suffers from severe activation collapse in deep layers, with activation standard deviations falling to $0.020$ (WA) and $0.017$ (TA). Oracle, Fed-BNC, and CC-Fed-BNC successfully realign the activation scales, bringing standard deviations back to the $0.035$--$0.067$ range. Crucially, under standard IID conditions, our privacy-preserving \textbf{Fed-BNC} ($25.89\%$) and \textbf{CC-Fed-BNC} ($26.82\%$) achieve full parity with the centralized \textbf{Oracle} ($26.29\%$). This proves that decentralizing the calibration process does not introduce statistical degradation or bias, and that 1D activation averages carry sufficient statistical information to realign representation scales without sharing raw data.

\subsection{The Representation Incoherence vs. Calibration Trade-off}
\label{subsec:tradeoff}
A surprising and significant result in \cref{tab:main_results} is that while calibration successfully realigns activation scales, it actually \textit{degrades} final classification accuracy (WA: $37.50\% \to 25.89\%$; TA: $27.99\% \to 15.51\%$). This reveals a critical and previously overlooked phenomenon: the \textbf{Representation Incoherence vs. Calibration Trade-off}. When merging experts fine-tuned on wildly disparate tasks (MNIST vs. FMNIST vs. CIFAR-10), the underlying parameter updates are incoherent and non-orthogonal, producing noisy feature maps. Under default uncalibrated merging, the activation collapse acts as an implicit, high-entropy regularization, damping down incoherent, noisy signals. However, when we perform BatchNorm calibration, we scale up the running variance to match these incoherent features, which amplifies the structural representation noise in deep layers and degrades accuracy. Post-merge calibration is highly effective for models on related tasks, but acts as a noise amplifier when merging models on highly disjoint domains.

\subsection{Verification of the Trade-Off in Coherent Settings}
\label{subsec:coherent_verification}
To verify our hypothesis, we execute a secondary experiment under a coherent task setting by splitting CIFAR-10 into three disjoint partitions, training separate expert models, and merging them. Since the experts share the same semantic task, their parameter updates are highly coherent. As shown in Figure~\ref{fig:coherent_results}, uncalibrated Weight Averaging performs exceptionally well, achieving $57.66\%$ accuracy (close to the expert average of $60.97\%$), despite severe activation collapse ($\sigma_{act} \approx 0.02$). This reveals that because the classification head is scale-invariant under $\operatorname{argmax}$, the network maintains high accuracy even with collapsed features. Crucially, when applying post-merge calibration via Oracle ($54.49\%$) or Fed-BNC ($47.13\%$), the performance gap is far smaller than in the disjoint setting, and the representation remains stable. This confirms that calibration behaves as expected (avoiding noise amplification) when the underlying merged weights are semantically coherent, validating our core hypothesis.

\subsection{Scientific Inquiry 1: Calibration Sample Efficiency}
We study the impact of local client sample sizes ($N \in [16, 512]$) on calibration. As shown in \cref{fig:sample_efficiency}, both Fed-BNC and CC-Fed-BNC are exceptionally sample-efficient. Under WA, with as few as $16$ samples, Fed-BNC achieves $25.83\%$ average accuracy and CC-Fed-BNC achieves $24.88\%$, remarkably close to the performance with $512$ samples ($26.27\%$ and $25.48\%$). This demonstrates that stable activation statistics converge extremely rapidly. Under the hood, because the spatial dimensions $H \times W$ of convolutional layers act as natural statistical replicators, a single image forward pass provides $H \times W$ activation samples per channel (e.g., $32 \times 32 = 1024$ samples per layer in early ResNet stages). This "Spatial Resolution Advantage" allows clients to compute highly accurate estimates with a minimal number of raw input samples, making Fed-BNC exceptionally practical for edge deployments.

\subsection{Scientific Inquiry 2: Robustness to Non-IID Class Skew}
In real-world federated and decentralized settings, data heterogeneity across clients is an omnipresent bottleneck. To simulate severe non-IID conditions, we restrict clients to observe only classes 0--4 (a extreme 50\% class skew) during the calibration forward pass.

As shown in \cref{tab:skew_results} and \cref{fig:non_iid_robustness}, naive Fed-BNC suffers severely under class skew, as its global statistics become heavily biased. In contrast, our proposed \textbf{CC-Fed-BNC} successfully mitigates this skew by collecting class-conditional statistics and performing class-balanced server aggregation. CC-Fed-BNC yields a powerful \textbf{$+1.73\%$} gain under WA (boosting accuracy from $22.77\%$ to $24.50\%$) and a \textbf{$+1.48\%$} gain under TA (boosting accuracy from $14.51\%$ to $15.99\%$) over naive calibration. This confirms that partition-based tracking of activation statistics prevents dominant or biased classes from distorting the global representation scales, ensuring robustness to extreme client-side skew.

\subsection{Scientific Inquiry 3: Robustness to Test-Time Covariate Shift}
We evaluate the robustness of our calibrated models under test-time Gaussian noise corruptions of varying intensities ($\sigma \in [0.0, 0.3]$) on CIFAR-10. This simulation represents real-world sensor degradation, lighting changes, or physical noise in physical deployments.

As displayed in \cref{fig:covariate_robustness}, all models decay gracefully. Fed-BNC maintains stable performance relative to the Oracle and uncalibrated models across all noise levels. Under the TA paradigm, at $\sigma=0.1$ noise, the uncalibrated model achieves $25.77\%$ accuracy, while the Oracle achieves $13.75\%$ and Fed-BNC achieves $15.00\%$. At a high noise level of $\sigma=0.3$, the uncalibrated model scores $22.16\%$, while the Oracle is at $13.55\%$ and Fed-BNC is at $14.28\%$. This shows that our decentralized statistical alignment does not introduce instability or sensitivity to real-world corruptions, proving that Fed-BNC is highly reliable in fluctuating physical environments.

\subsection{Scientific Inquiry 4: Federated Quantization Calibration (Fed-QC)}
To optimize for edge serving, Post-Training Quantization (PTQ) to INT8 is standard practice, reducing model size by $4\times$ and speeding up inference. However, quantizing a merged model requires a calibration dataset to set activation observers. Under strict privacy frameworks, centralizing raw user data is impossible. To address this, we propose \textbf{Federated Quantization Calibration (Fed-QC)}: the server broadcasts a quantization-ready model, clients run a forward pass on their private local datasets ($N\!=\!128$) to record min/max activation ranges, and transmit only these 1D statistical boundaries. The server averages these ranges to calibrate global observers and converts the model to INT8.

We evaluate Fed-QC against FP32 baselines, naive uncalibrated INT8 (no forward pass), and centralized INT8 Oracle (using $N\!=\!1024$ central samples). As displayed in \cref{tab:quantization_results}, naive INT8 quantization triggers catastrophic representation collapse, dropping accuracies to random chance ($\sim 9.5\%$). Crucially, both central Oracle and our decentralized \textbf{Fed-QC} completely restore performance. Under Weight Averaging, Fed-QC achieves a remarkable \textbf{$36.18\%$} average accuracy, recovering to within $1.32\%$ of FP32 ($37.50\%$) and actually outperforming the central Oracle by $+0.56\%$. Under Task Arithmetic, Fed-QC achieves \textbf{$26.91\%$} average accuracy, matching the central Oracle within $0.24\%$ and recovering to within $1.08\%$ of FP32 ($27.99\%$). This proves that model merging and Post-Training Quantization can be successfully combined in a fully decentralized, low-bandwidth, and privacy-preserving manner.

\subsection{Servability, Latency \& Compiler Optimizations}
From a deployment perspective, Fed-BNC is exceptionally practical: it modifies only the 1D running buffers of existing BatchNorm layers, adding exactly 0 parameters, 0 server-side optimization, and 0 runtime latency. Additionally, because Fed-BNC does not require active test-time hooks (unlike streaming calibration), the resulting model remains fully static. We test compiling our calibrated model using \texttt{torch.compile}. Evaluated on a batch size of 64 on an H100 GPU, the compiled model runs flawlessly, achieving an average latency of \textbf{18.89 ms} compared to \textbf{19.16 ms} in native eager mode, with zero compilation or runtime errors.

\subsection{Guidelines and Recommendations for Real-World Deployments}
\label{subsec:guidelines}
Based on our findings, we propose the following actionable guidelines for production edge deployments:
\begin{itemize}
    \item \textbf{Assess Representation Coherence:} Calibration is highly effective for semantically related tasks, but acts as a noise amplifier on incoherent, disjoint task merges.
    \item \textbf{Use CC-Fed-BNC by Default:} Real-world client data is rarely IID. CC-Fed-BNC eliminates local class biases under extreme skew while adding less than $350\text{ KB}$ of bandwidth overhead.
    \item \textbf{Leverage Small Budgets ($N\!=\!32$--$64$):} Due to the Spatial Resolution Advantage (spatial dimensions act as statistical replicators), activation statistics converge rapidly. Small sample sizes are sufficient.
    \item \textbf{Compile the Final Model:} Because Fed-BNC produces a static network, practitioners should always run \texttt{torch.compile()} to maximize serving throughput on production hardware.
\end{itemize}

\section{Conclusion}
\label{sec:conclusion}
We proposed Federated BatchNorm Calibration (Fed-BNC) and Class-Conditional Federated BatchNorm Calibration (CC-Fed-BNC) for multi-task model merging. By collecting and aggregating only 1D activation statistics from client-side forward passes, our approach achieves central-oracle-level representation alignment with zero server data access and absolute client privacy. It is exceptionally sample-efficient, robust to non-IID class skew, and completely compatible with high-performance production compilers. Finally, we identified the Representation Incoherence vs. Calibration Trade-off, providing key theoretical and practical guidelines for future research in multi-task model merging.
